# 01_d 文風樣本與指紋

若本文件與舊聊天紀錄衝突，以本文件為準。

本文件是 Template repo 的文風錨定通用骨架，不綁定任何特定作品。
本文件中的章節結構與 placeholder 只示範 schema，不代表任何作品的最終指紋。
Instance bootstrap 與上游協議完成、corpus 累積到一定量後，才可由人類或 `/anchor-style`（如未來實作）填入作品實際指紋與樣本。

---

# 0. 使用方式

本文件分成 5 個章節：

1. `# 1` 三層敘事結構比例 — 全局敘事層比例硬約束
2. `# 2` 全局文風指紋 — 全角色合計的句長 / 標點 / 旁白模式
3. `# 3` Per 角色聲線樣本集 — 各主要角色的量化指紋 + 正例 + 負例（每角色一個子節）
4. `# 4` `/scene-task` 抽取規則 — 下游 `/scene-task` 從本檔抽指紋進任務包 §18.4 的規則
5. `# 5` 維護規則 — 指紋校準頻率 / corpus 擴張時的重抽流程

Instance 寫入時必須含完整 5 章節結構，不得刪減；具體內容由作品累積 corpus 後依本提案 `_design/STYLE_ANCHOR_PROPOSAL.md` v0.1 §4 §5 §6 拷貝寫入（RFC §6 內使用「樣本師」一詞，本檔統一改用「樣本集」；instance 端寫入時亦使用「樣本集」）。

---

# 1. 三層敘事結構比例

**Template 端佔位說明（待 instance 寫入）：** 本節由 instance 端依作品 corpus 分析結果寫入。

寫入時應包含：
- 對話層「」字數 + 比例
- 旁白層（）字數 + 比例
- 其他（無標記）字數 + 比例
- 各層主要負責角色 + 旁白歸屬規則
- 旁白層佔比成為作品辨識特徵的程度說明

（未來若作品需要全知敘述者層，另立 01_e 或在本節加第四層。）

---

# 2. 全局文風指紋

**Template 端佔位說明（待 instance 寫入）：** 本節由 instance 端依作品 corpus 分析結果寫入。

寫入時應包含：
- 主角雙層（對話 + 旁白）特殊性（若主角同時負責旁白層）
- 句長分佈 per 角色（mean / median / min / max / 分佈級距）
- 標點密度 per 角色（軟停 / 硬停 / 省略 / 破折）
- 旁白模式（視角 / 內容類型 / 動作描述邊界 / 句長 / 特例）
- 跨角色禁用句型（如零破折號等）

---

# 3. Per 角色聲線樣本集

**Template 端佔位說明（待 instance 寫入）：** 本章節由 instance 端依各主要角色 corpus 抽取的指紋寫入。每個主要角色一個 `## 3.N <角色名>` 子節。

每個角色子節應含 3 個 subsection：
- `### 量化指紋` — 句長 / 標點密度 / 特徵詞 / 旁白特性
- `### 正例樣本` — 從 corpus 抽 3-5 條對話正例 + 3-5 條旁白正例（如該角色負責旁白）
- `### 負例樣本` — 反模式範例 3 條對話 + 3 條旁白（design-time 寫的反模式）

---

# 4. `/scene-task` 抽取規則

**Template 端佔位（已 spec；instance 端不必改寫，依下游 skill 行為對齊）：**

下游 `/scene-task` 階段 2 探索時，從本檔抽取以下進任務包 §18.4 文風錨定：

1. 本場出場角色 → 對應 `## 3.N` 子節
2. 各出場角色「量化指紋」全條（句長 / 標點 / 特徵詞）
3. 各出場角色「正例樣本」前 3 句
4. 各出場角色「負例樣本」全部（反模式提醒密度高）
5. 全作品「跨層硬約束」（從 `# 1` 抽，如零破折號 / 旁白歸屬等）

抽取時保持 source-labeled，禁複製整個 01_d 內容進任務包。

---

# 5. 維護規則

**Template 端佔位（已 spec；instance 端可依作品實際情況調整參數）：**

1. **指紋校準頻率：** 每累積 5 個新 Level（或 user 自訂閾值）重跑 `_tools/fingerprint_analyzer.py`（如未來實作）校準。偏差 > 15% 須拍板是否更新基線。
2. **不擅自漂移：** 後期作品弧線推進、角色關係改變時聲線可能合理漂移；若漂移已成新基線，於本檔加「階段聲線變化」段（對齊 03_b §17.1）。
3. **負例累積：** 真實生成失敗的 LLM 預設輸出可作為負例補入（標明來源 + 失敗點）。
4. **新角色加入：** 主要角色新登場且 corpus ≥ 1 個 Level 後，補入 `# 3` 新子節。
