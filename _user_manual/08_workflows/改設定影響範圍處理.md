# 情境 4：改設定 + 影響範圍處理

> ⏳ **本文為骨架**，待 Phase C 完成後補真實對話。

# 1. 適合情境

你寫到第 5 章發現要改第 1 章的角色設定 / 世界規則 / 主線方向。

# 2. 預估工時

**單一設定改：1-2 小時**（含找出受影響檔 + 重審）。

# 3. 流程

## Step 1：找出要改的設定

用 F6 搜尋找到相關檔。例：找「主角A」找到聲線卡 + 關係矩陣 + 弧線表。

## Step 2：跑 /iterate-* 改設定

如：

```
/iterate-character C-主角A
```

agent 跟你對話只動相關議題（不重跑整套 9 議題）。

## Step 3：看影響範圍掃描

agent 改完後印：

```
⚠ 以下 24 個檔依賴 C-主角A：

LOCKED 檔（3 個）— 動不了，要走降級流程：
- 08_dialogue_outputs/CH01_S03_v02.md
- 08_dialogue_outputs/CH01_S05_v02.md
- ...

FINAL 檔（18 個）— 可重審：
- ...

DRAFT 檔（3 個）— 無影響：
- ...

[📋 複製重審清單到剪貼簿]
```

## Step 4：處理 LOCKED 檔（降級流程）

對每個 LOCKED 檔：

1. 走 [11 故障排除](../11_troubleshooting.md) §5.1 降級流程
2. 改 `狀態：LOCKED → DEPRECATED`
3. 在 09_e final-gating 紀錄補一條降級紀錄
4. 重做新版本：跑 `/dialogue-write` 從 trial 重來 / 或 `/dialogue-write --single-iter` 局部迭代

## Step 5：處理 FINAL 檔

對每個 FINAL 檔：

1. 重跑 `/qa` 確認是否仍 PASS
2. 如果 FAIL → 修補或重寫
3. PASS → 維持 FINAL

## Step 6：DRAFT 檔不用動

下次寫到時 agent 自動讀新設定。

# 4. 反查機制原理

工具靠 frontmatter `depends_on` 反查：

```yaml
# 08_dialogue_outputs/CH01_S03_v02.md
---
entities: [S-01-03]
depends_on:
  - C-主角A          ← 依賴主角A
  - C-反派B
  - W-rules
---
```

當你改 C-主角A → 工具掃所有 `depends_on: C-主角A` 的檔 → 列出來。
