# 情境 2：從手稿導入建專案

> ⏳ **本文為骨架**，待 Phase A.0 + B 完成後補真實對話 / 截圖。

# 1. 適合對象

**你已經在外部寫好設定**，想快速導入工具結構化管理：

- 在 Word / Google Docs 寫好世界觀
- 在 ChatGPT / Claude 對話產出主角設定
- 在 Notion 寫好章節大綱
- ...

# 2. 預估工時

- 手稿導入 4 個（世界觀 / 角色 / 主線 / 細綱）：**2-4 小時**
- 比情境 1 快 10 倍

# 3. 前置條件

**手稿必須有 markdown structure**（`#` / `##` 段落分割）。

```
✅ 好的手稿（agent 能解析）：
# 魔法學院世界觀
## 世界類型
高魔幻
## 世界規則
- 魔法來源：契約制
- 代價：每用扣壽命 1 天
...

❌ 不好的手稿（agent 拒絕）：
這是一個高魔幻的世界，背景是現代地球...
（純文字無段落分割）
```

# 4. Step-by-step

## Step 1：準備手稿

把外部手稿整理成 markdown structure。每個 `##` 對應一個議題。

## Step 2：啟動 /create-world

跑：

```
/create-world
```

agent 印階段 1 開場後，**貼整段手稿** + 觸發語：

```
[你的手稿全文]

直接寫檔 --trust-level=agent_assisted
```

## Step 3：agent 解析 + 補洞

agent 自動：

1. 識別手稿的 `##` 段落
2. 對應到 11 議題清單
3. 列出**缺漏的議題**（REQUIRED 必補；STRONGLY_PREFERRED 標 TODO）
4. 若有衝突（既有 entity 同名）→ 跳 4 選項 modal

REQUIRED 議題缺漏：補答後重貼。

## Step 4：寫檔

agent 自動拆分到對應檔案：

- `01_world/01_a_世界觀總覽.md`
- `01_world/01_b_世界語言規格.md`
- `01_world/01_c_陣營與階級語言.md`
- `02_vocabulary/02_a_可用詞.md`
- `02_vocabulary/02_b_禁用詞.md`
- `02_vocabulary/02_c_慎用詞.md`
- `00_protocol/00_b §1 §2 §10`

phase_log 紀錄 `import_source: agent_assisted`。

## Step 5：重複 step 1-4 對角色 / 主線 / 細綱

- `/create-character` 每個重要角色
- `/create-relationship` 每對重要關係
- `/create-outline` 主線
- `/create-detailed-outline` 章節 + 場景索引

---

# 5. trust-level 差異（v1.1 對齊）

| trust-level | 適用 | 對下游影響 |
|---|---|---|
| `agent_assisted` | 手稿在外部 agent（CC / Cowork）編輯過 | **無**（下游永遠跑完整 8 份 QA）|
| `external_llm` | 手稿從外部 LLM（ChatGPT 等）直接得到 | **無**（同上）|

⚠ **trust-level 只記錄資料來源，不影響下游 pipeline**（v1.1 master 第四輪 CC-06 校正）。

---

# 6. 衝突處理 4 選項

如果手稿中的 entity 跟既有 entity 同名 → 跳 conflict modal：

| 選項 | 行為 |
|---|---|
| (1) merge | 把手稿併入既有 |
| (2) overwrite | 用手稿覆蓋既有 |
| (3) create-as-new | 手稿另存為新 ID（如 `C-主角A_v2`）|
| (4) skip | 拒絕導入該 entity |

詳見 [11 故障排除](../11_troubleshooting.md) §5.3。
