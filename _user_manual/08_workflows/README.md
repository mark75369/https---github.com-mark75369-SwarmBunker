# 08 工作流情境

> ⏳ **本章為情境骨架**，待各 phase 完成後補真實對話範例 / 截圖。  
> 目前提供 5 個情境的步驟綱要。

# 1. 情境清單

| # | 情境 | 適合誰 | 預估工時 | 待補狀態 |
|---|---|---|---|---|
| 1 | [從零開始建專案](從零開始建專案.md) | 純新作 / 從零開始的 user | 2-3 天密集 | ⏳ Phase A-C 完成後補真實對話 |
| 2 | [從手稿導入建專案](從手稿導入建專案.md) | 已有 Word / GDoc 設定的 user | 2-4 小時 | ⏳ Phase A.0 完成後補範例 |
| 3 | [寫單場戲完整流程](寫單場戲完整流程.md) | 日常 90% 工作 | 30-60 分鐘 / 場 | ⏳ Phase D 完成後補 |
| 4 | [改設定影響範圍處理](改設定影響範圍處理.md) | 寫到後期想改前期設定 | 1-2 小時 | ⏳ Phase C 完成後補 |
| 5 | [交付到下游](交付到下游.md) | 對 Unity 工程師 / 翻譯 / 配音 | 30 分鐘 | ⏳ Phase D 完成後補 |

# 2. 推薦學習順序

| 你的狀況 | 建議看哪個 |
|---|---|
| 第一次用 | 看 [00 快速入門](../00_quick_start.md)（10 分鐘）|
| 想開新專案 | 看情境 1 或 2 |
| 已建好 Bible 要寫對白 | 看情境 3 |
| 設定要修 | 看情境 4 |
| 要交給工程師 / 翻譯 | 看情境 5 |
