# 00_g 大綱創建協議

## 1. 文件目的

本協議規範 `/create-outline` / `/建立大綱` 在主線實體創建過程中的執行流程。

本協議用於建立 `P` 實體，並引導使用者形成：

- 主線一句話
- 核心衝突
- 主題
- 想要 vs 需要
- 結構選擇
- 與世界規則、角色聲線的合規檢查
- 規模定位
- 類型偏移風險清單

本協議只處理主線骨架，不寫章節細綱、不寫場景索引、不寫任務包、不寫正式台詞。

完成後主要寫入：

- `05_plot/05_a_主線大綱模板.md`
- `05_plot/05_b_章節結構模板.md` 的章節空殼
- `00_protocol/00_b_反ai味檢查表.md` §3 與 §4

## 2. 啟動條件

使用者以 `/create-outline` 或 `/建立大綱` 啟動本流程時，agent 必須先確認：

| 條件 | 要求 | 不符合時處理 |
|---|---|---|
| 世界觀規則 | `W-rules` 至少 REVIEW | 啟動拒絕，或走 `prereq_waived: true` 共通流程 |
| 詞彙系統 | `V` 至少 REVIEW | 啟動拒絕，或走 `prereq_waived: true` 共通流程 |
| 語言規格 | `W-language` 至少 REVIEW | 啟動拒絕，或走 `prereq_waived: true` 共通流程 |
| 主角 | 至少有主角 `C-*` 處於 DRAFT 或以上 | 強烈建議先跑角色創建；可 waiver |
| 寫入狀態 | 目標檔不是 LOCKED，或已有明確授權 | 若 LOCKED，停止並請使用者確認 |
| issue registry | `<instance_root>/issue_type_registry.yaml` 可讀；若不存在，從 `_design/registries/issue_type_registry.template.yaml` Template fallback + WARN；對應 skill key `00_g_outline` 必須存在 | YAML parse error、缺 `00_g_outline` skill key、議題 id 違規、locked 欄非 bool 等 schema 異常時，拒絕進階段 2 |

若主角缺而使用者要求繼續，agent 必須印：

> 警告：主線設計沒有主角作為依據，戲劇邏輯可能架空。

## 3. 階段 1：自由輸入診斷

### 3.1 開場提示

agent 啟動後必須先說：

> 主線是這個作品的骨幹。請貼出你對主線的長段想法，至少包含一句話定位、核心衝突、想表達的主題、開頭結尾、可能的章節節奏。如果你還在發散階段，貼草稿也行，我會幫你診斷哪些已成型、哪些是空的。

### 3.2 診斷重點

使用者貼完後，agent 依 `00_protocol/00_a_台詞生產協議.md` §3.3 診斷模式輸出診斷報告。

診斷至少包含：

| 診斷項 | 檢查內容 |
|---|---|
| 主線清晰度 | 主角、想要、阻力是否在主線一句話中可見 |
| 想要 vs 需要 | 主角表層目標與內在缺口是否已隱含 |
| 結構承載力 | 目前構想是否能撐到使用者預期長度 |
| 世界觀合規 | 是否違反 W-rules 或 V |
| 角色合規 | 是否強迫主角做出聲線卡禁止的行為 |
| 類型偏移 | 是否已出現 00_b 禁止的方向 |

### 3.3 診斷結尾

診斷報告結尾必須預告階段 2 會跑 registry core 預設 6 個 user-facing 議題；最終議題清單依 `issue_type_registry.yaml` 的 `user_extensions` / `core_overrides` 動態構建：

1. 必先三件事
2. 想要 vs 需要測試
3. 結構選擇
4. 與世界規則、角色合規性檢查
5. 規模定位
6. 類型偏移風險清單

拆分規則屬 §10.7 agent-side execution mechanic，不列入 registry user-facing 議題。

## 4. 階段 2：逐題探索順序

## 4.0 議題清單動態構建（D-047 / Contract D §4a.2）

階段 2 啟動時，agent **必須**讀 `<instance_root>/issue_type_registry.yaml` 的 `00_g_outline` skill key 區段，按 **`core + user_extensions − core_overrides`** 動態構建議題清單：

1. **core 議題**先依 `id` 升序排列（id 1-99 為 core 保留範圍）。
2. **`user_extensions[*]`** 接在 core 議題後（id ≥ 100 為 user 自訂議題保留範圍）。
3. 套用 **`core_overrides[*].skip_id`** 過濾：
   - 對應 core entry `locked=true` → agent **忽略**該 override 並 **WARN**「locked 議題 id={n} 不可 SKIP，core_overrides 條目被忽略」；該議題仍出現在清單
   - 對應 core entry `locked=false` → 從議題清單中移除
   - `skip_id` 不存在於 core → WARN「core_overrides skip_id={n} 找不到對應 core 議題；條目被忽略」
4. 對每個議題：
   - `required_level=REQUIRED` → 跳階段 4 前**必答**；缺漏拒絕進階段 4
   - `required_level=STRONGLY_PREFERRED` → 跳階段 4 允許但 phase_log 紀錄缺漏 + 對應分拆檔段落標 TODO
   - `required_level=OPTIONAL` → 跳階段 4 允許不紀錄
5. `core[*].question_summary` 只作為 opener；agent 完整提問腳本詳本檔 §10 對應段（沿用 UD §1.3.2 完整內容）。
6. **拆分規則不入 registry** — 屬 agent 階段 4 寫檔 mechanic，由 §10.7 固定執行，不對 user 提問。

**Registry 異常處理：**
- 若 registry 載入失敗（YAML parse error / 缺 `00_g_outline` key 等）→ agent **拒絕進階段 2**，請 user 修補 registry
- 若 `user_extensions[*].id < 100` 或 與 core id 重複 → parser ERROR；agent 拒絕進階段 2

## 4.1 議題順序預設表

00_g core 共 **6 個 user-facing 議題**（registry 對應 id 1-6）。若 user 在 `user_extensions:` 加自訂議題，按上述 §4.0 規則接於 core 後。

| 順序 | 議題 | required_level | locked | 為什麼這時問 |
|---|---|---|---|---|
| 1 | 必先三件事 | REQUIRED | true | 主線一句話、核心衝突與主題是 P 主檔最小骨架。 |
| 2 | 想要 vs 需要測試（戲劇經典原則） | REQUIRED | true | 主角弧線根基會影響角色動機、高潮選擇與下游判定。 |
| 3 | 結構選擇 | REQUIRED | true | 幕數與章節範圍會決定 05_b 章節空殼與後續細綱規模。 |
| 4 | 與世界規則、角色合規性檢查 | STRONGLY_PREFERRED | false | 先攔截主線與 W-rules 或 C-* 聲線的衝突。 |
| 5 | 規模定位 | STRONGLY_PREFERRED | false | 敘事賭注規模會寫入 00_b §3，供 09_f 判定。 |
| 6 | 類型偏移風險清單 | STRONGLY_PREFERRED | false | 先列出不允許走的方向，避免後續主線偏移。 |

**§10.7 拆分規則**屬 agent-side execution mechanic（非 user-facing 議題），階段 3 收斂預告稿時固定列入寫檔計畫；user 只需在預告稿確認，不對拆分規則本身提問。詳本檔 §10.7。

agent 必須依 §4.0 動態構建後的清單提問，不得跳到拆分或寫檔。

每題都必須輸出：

- 為什麼問
- agent 怎麼問
- 使用者預期答什麼
- agent 怎麼整理寫檔
- 拒答 / 跳題處理

## 5. 階段 3：收斂與寫入預覽

完成階段 2 後，agent 必須輸出收斂預告稿並等待使用者確認。

收斂預告稿必須包含：

| 項目 | 要求 |
|---|---|
| 主線一句話最終版 | 請使用者逐字確認 |
| 核心衝突一句話 | 請使用者逐字確認 |
| 主題一句話 | 請使用者逐字確認 |
| 想要 vs 需要 | 顯示三子題整理結果 |
| 結構預覽 | 幕數、章節數範圍與章節空殼 |
| 00_b §3 預覽 | 規模定位將被覆寫的內容 |
| 00_b §4 預覽 | 類型偏移風險清單將被覆寫的內容 |

未經使用者明確確認，agent 不得進入階段 4。

## 6. 階段 4：拆分執行

使用者確認後，agent 才可依區段 10.7 寫入。

固定寫入順序：

1. `05_plot/05_a_主線大綱模板.md`
2. `05_plot/05_b_章節結構模板.md` 的章節空殼
3. `00_protocol/00_b_反ai味檢查表.md` §3 規模定位
4. `00_protocol/00_b_反ai味檢查表.md` §4 類型偏移風險清單

本階段不得寫章節細節、場景索引、場景任務包或正式台詞。

## 7. 階段 5：驗證與狀態回報

拆分完成後，agent 必須回報 `/status` 預期變化：

| 項目 | 預期 |
|---|---|
| P | 新增為 DRAFT，完成度至少 25% |
| CH-NN | 若建立章節空殼，完成度約 10% |

| 05_b | 只含章節空殼，不含細綱內容 |

- P 經人類審後升 REVIEW
- 進入細綱創建流程

## 8. 禁止事項

本協議執行期間禁止：

1. 寫章節細綱。
2. 寫場景索引或場景任務包。
3. 寫具體台詞片段。
4. 擅自把使用者沒講的次要角色當作主角納入主線。
5. 用主線大綱替角色補聲線卡。
6. 擅自修改 W-rules、V、W-language 或角色聲線卡核心內容。
7. 把 P 自動升 REVIEW。
8. 對 LOCKED 目標檔直接覆寫。

## 10. 專屬議題腳本

### 10.1 必先三件事

| 欄位 | 內容 |
|---|---|
| 為什麼問 | 主線骨架最小集，缺一不可。 |
| agent 怎麼問 | 主線的最小集三題，每題一句話：a. 主線一句話，含主角、想要、阻力。b. 核心衝突，含想做的事 vs 阻止他的力量。c. 主題，形式是疑問或主張，不是道德教訓。 |
| 使用者預期答什麼 | 三子題各一句話。 |
| agent 怎麼整理寫檔 | 寫入 `05_a §1 必先三件事` 三個子節；若三句話邏輯矛盾，標 `CONFLICT(三件事不一致)` 並回階段 2。 |
| 拒答 / 跳題 | 任一缺，拒絕進階段 3。 |

### 10.2 想要 vs 需要測試

| 欄位 | 內容 |
|---|---|
| 為什麼問 | 主角弧線根基；缺了會影響 09_b 與 09_d 判定。 |
| agent 怎麼問 | 三題：a. 主角想要什麼，表層且具體。b. 主角需要什麼，內在缺口或成長。c. 兩者如何衝突，高潮處主角面臨什麼選擇。 |
| 使用者預期答什麼 | 三子題各一段。 |
| agent 怎麼整理寫檔 | 寫入 `05_a §2 想要 vs 需要`；若想要與需要沒有衝突，標 INFERENCE 提醒；可補到主角聲線卡的弧線階段作待確認註記。 |
| 拒答 / 跳題 | 強烈建議不跳；若跳，05_a §2 標 TODO。 |

### 10.3 結構選擇

| 欄位 | 內容 |
|---|---|
| 為什麼問 | 章節數估計影響 00_h 細綱與整個 pipeline 規模。 |
| agent 怎麼問 | 兩題：a. 幕數結構，三幕、四幕、五幕、多幕、單章長篇、片段集合或自訂。b. 章節數估計，給範圍即可。 |
| 使用者預期答什麼 | 兩子題各一句。 |
| agent 怎麼整理寫檔 | 寫入 `05_a §3 結構`；依幕數估算章節區間，請使用者確認；確認後在 `05_b` 建立每章空殼。 |
| 拒答 / 跳題 | 可跳；跳了 05_b 不建立骨架，細綱流程會缺先決。 |

### 10.4 與世界規則、角色合規性檢查

| 欄位 | 內容 |
|---|---|
| 為什麼問 | 主線若違反 W-rules 或主角聲線，下游會連鎖失準。 |
| agent 怎麼問 | agent 用主線一句話、想要 vs 需要、結構跑兩層檢查：主線 vs W-rules；主角弧線 vs `C-*` 聲線。請使用者逐項確認或修正。 |
| 使用者預期答什麼 | 對初判逐項回應 OK 或修正。 |
| agent 怎麼整理寫檔 | 寫入 `05_a §4 合規檢查紀錄`；若衝突，標 CONFLICT，並請使用者決定修主線、修 W-rules 或修聲線。 |
| 拒答 / 跳題 | 跳過時在 05_a 標警告。 |

### 10.5 規模定位

| 欄位 | 內容 |
|---|---|
| 為什麼問 | 決定整體敘事尺度，避免後續被寫成不符規模的故事。 |
| agent 怎麼問 | 本作賭注規模屬於哪一類，可組合並排序：對世界要命、對角色要命但對世界不重要、對特定群體要命、混合 / 其他。 |
| 使用者預期答什麼 | 排序選擇與一段說明。 |
| agent 怎麼整理寫檔 | 寫入 `00_b §3 規模定位` 與 `05_a §5 規模定位`；並反推禁止寫法，加入類型偏移風險候選。 |
| 拒答 / 跳題 | 強烈建議不跳；若跳，09_f 缺重要基線。 |

### 10.6 類型偏移風險清單

| 欄位 | 內容 |
|---|---|
| 為什麼問 | 提前列出本作不允許走的方向，供 09_f 判定。 |
| agent 怎麼問 | 請列 5-10 條本作不允許走的方向，可從巨大陰謀化、單一反派化、救世主敘事、皆大歡喜結尾、死亡煽情化等方向發想。 |
| 使用者預期答什麼 | 5-10 條具體禁區。 |
| agent 怎麼整理寫檔 | 寫入 `00_b §4 類型偏移風險清單` 與 `05_a §6 類型偏移風險清單`；若主線一句話含明文禁止方向，標 CONFLICT。 |
| 拒答 / 跳題 | 強烈建議不跳；若跳，09_f 只能依規模定位粗判。 |

### 10.7 拆分規則

| 來源議題 | 目標 | 寫入方式 |
|---|---|---|
| 10.1 必先三件事 | `05_a §1` 三子節 | 覆寫 |
| 10.2 想要 vs 需要 | `05_a §2` 三子節 | 覆寫 |
| 10.3 結構選擇 | `05_a §3` + `05_b` 章節空殼 | 覆寫 + 建立 |
| 10.4 合規檢查 | `05_a §4` | 覆寫 |
| 10.5 規模定位 | `05_a §5` + `00_b §3` | 覆寫 + 覆寫附時間戳 |
| 10.6 類型偏移風險 | `05_a §6` + `00_b §4` | 覆寫 + 覆寫附時間戳 |

寫檔順序固定為：

1. `05_a`
2. `05_b` 章節空殼
3. `00_b §3`
4. `00_b §4`

### 10.8 Frontmatter 規範

本協議自身通常不貢獻實體，可省略 YAML block。

依本協議寫入資料檔時：

| 目標 | entities | depends_on | weight |
|---|---|---|---|
| 05_a | `[P]` | `[W-rules, V, W-language, <主角 C-*>]` | `{P: 1.0}` |
| 05_b 章節空殼 | `[CH-01, CH-02, ...]` | `[P]` | `{CH-NN: 0.1}` |

章節空殼只代表下游可展開；正式章節內容由細綱流程補足。

## 11. 下游銜接點

完成本協議後：

- `P` 實體完成度至少 25%。
- 每個章節 `CH-NN` 可出現為約 10% 的空殼。

- `00_b §3` 與 `00_b §4` 會成為 09_f 類型偏移檢查的核心基線。
