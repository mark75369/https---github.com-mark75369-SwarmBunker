# 09_i 跨場一致性檢查模板（通用版）

## 用途

本模板用於檢查多場台詞檔之間的跨場一致性。

跨場一致性不是新增獨立 continuity entity，而是在 `/qa` pipeline 中檢查同一角色、同一資訊、同一章節節奏與伏筆回收在多場之間是否連續、是否提前、是否互相矛盾。

本模板是通用 Template，不含任何作品專屬內容。Instance 執行 `/qa` 時，才依該作品的場景索引、角色聲線卡、資訊揭露表、伏筆回收表與台詞檔填入實際檢查對象。

## qa_type

本模板對應 `qa_type_registry` 的 enum 值：

```yaml
qa_type: CROSS_SCENE_CONTINUITY
```

`CROSS_SCENE_CONTINUITY` 是 8 份 `/qa` 必跑報告之一。依 SPEC §12.7 的序列，09_i 最後印出，因為它需要參照 09_b、09_d、09_g、09_h 的場級結果做跨場比對。

## 5 層檢查框架

### 1. 檢查範圍定位

- 讀取目標台詞檔。
- 從 `source_task` 反查任務包。
- 讀取本場 `entities` / `depends_on`。
- 決定 `cross_scene_scope`：chapter / arc / all。
- 排除 DRAFT、REVIEW、DEPRECATED、deleted 與本場本身。

### 2. 跨場資料抽取

- 反查範圍內所有已 FINAL / LOCKED 的相關場景。
- 抽取同一 C-* 的歷史台詞，用於聲線漂移比對。
- 抽取 05_d 資訊揭露表與 05_e 伏筆回收表。
- 抽取歷史 09_g / 09_h 報告或即時計算場級節奏與張力指標。

### 3. 三面向檢查

- `VOICE_DRIFT`：跨場聲線漂移。
- `INFO_LEAK`：跨場資訊洩漏、重複揭露與 implication 矛盾。
- `RHYTHM_ARC`：跨場節奏與張力 arc。

### 4. 與上游基準對齊

- 對照 05_c 角色弧線表。
- 對照 05_d 資訊揭露表。
- 對照 05_e 伏筆與回收表。
- 對照 05_b 章節節奏標籤。
- 對照角色聲線卡的跨階段變化規則。

### 5. 分級與輸出

- `PASS`：跨場聲線、資訊與節奏均能合理連續。
- `NEEDS_REVIEW`：跨場變化可能有設計意圖，但上游依據不足或需人類確認。
- `FAIL`：提前揭露、重複揭露、角色失憶、跨場聲線退化或章節節奏錯位。

## 檢查 algorithm

09_i 跑三個面向，每面向各自輸出子報告。

### 跨場範圍定義

09_i 預設必跑，不再由 opt-in 旗標啟動。`--scope` 旗標的作用是切換跨場範圍，不是 on/off 開關。

| 範圍代號 | 涵蓋 |
|---|---|
| `--scope=chapter`（預設） | 同 CH-NN 內已 FINAL / LOCKED 的場景 |
| `--scope=arc` | 涵蓋指定角色 arc 的所有章節，從 05_c 角色弧線表反查 |
| `--scope=all` | 全 Instance 已 FINAL / LOCKED 的場景，成熟期才有意義 |

跨場範圍不含：

- DRAFT / REVIEW 場，因為未定稿，本身會變。
- DEPRECATED / `status: deprecated|deleted` 場，因為已棄用。
- 本場本身，因為本場是要拿來與其他場比對的對象。

依 REQUIREMENTS_LOCK §4.1 的拍板語境，跨場 QA 的「跨」範圍可涵蓋跨章、跨集與主角 arc 全長；實際執行由 `cross_scene_scope` 控制。

### 面向 1：跨場聲線漂移（VOICE_DRIFT）

對被檢查場的每個出場 C-*：

1. 反查 `--scope` 內所有出場該 C-* 的場景（用 `entities` / `depends_on` 反查）。
2. 對每個歷史場提取該角色全部台詞句。
3. 計算句長 `μ` / `σ`、典型句型分布、慣用詞表（top 20 高頻詞）。
4. 與本場該 C-* 的同類指標比對：
   - 句長 `μ` 差 > 30%：標「跨場句長漂移」（NEEDS_REVIEW）。
   - 慣用詞 top 20 重疊 < 40%：標「跨場用詞漂移」（NEEDS_REVIEW）。
   - 角色聲線卡列出的禁用句型在本場出現但歷史場未出現：標「角色聲線退化」（FAIL）。

補充規則：

- 若 05_c 明示該角色在本場前後有弧線變化，聲線漂移不直接 FAIL，但需對照弧線階段。
- 若任務包明示角色偽裝、壓抑或策略性表演，標 NEEDS_REVIEW 並要求人類確認。
- 若本場漂移同時被 09_b 命中，09_i 需標示為跨場層級問題。

### 面向 2：跨場資訊洩漏（INFO_LEAK）

讀 05_d 資訊揭露表 + 05_e 伏筆與回收表：

1. 列出 `--scope` 內每個場景已揭露的資訊（從歷史場 09_d 報告反查，或從 05_d 標記欄位反查）。
2. 對本場所有明確陳述句做資訊提取。
3. 比對：
   - 本場揭露的資訊早於 05_d 規劃時間：標「資訊提前揭露」（FAIL），並與 09_d 報告交叉驗證。
   - 本場揭露的資訊已被歷史場揭露，但本場用未知 / 第一次語感：標「角色失憶 / 重複揭露」（FAIL）。
   - 本場 implication 與歷史場 implication 衝突：標「跨場 implication 矛盾」（NEEDS_REVIEW）。

補充規則：

- 若本場只是暗示，不得自動等同於揭露；需依 05_d 的「只能暗示」欄位判定。
- 若伏筆傳遞到本場但沒有承接，需對照 05_e 判定是否缺漏。
- 若本場新增資訊不在 05_d / 05_e，標「上游缺漏或 canon delta 候選」，不在本模板內擅自補寫設定。

### 面向 3：跨場節奏 arc（RHYTHM_ARC）

對 `--scope` 內所有場（含本場）：

1. 抽 09_g 報告（如歷史場有跑過 09_g），或即時計算 `μ`、`CV`、`tension_total`。
2. 按 `S-<ch>-<n>` 編號排序畫節奏曲線。
3. 判定：
   - 曲線全平坦（連續 ≥ 5 場 `μ` 與 `tension_total` 都接近）：標「全章節節奏單一」（NEEDS_REVIEW）。
   - 曲線無高潮（章節節奏 high 的場 `tension_total` 反而低於 mid / low 場）：標「章節結構失衡」（FAIL）。
   - 曲線與 05_b 章節節奏標籤完全反向：標「章節節奏錯位」（FAIL）。

補充規則：

- 09_i 不替代 09_g；09_g 負責單場節奏，09_i 只使用場級指標做跨場曲線。
- 09_i 不替代 09_h；09_h 負責單場攻防力度，09_i 只觀察多場張力走勢。
- 若缺少歷史 09_g / 09_h 報告，09_i 可即時計算必要指標，但需在報告中標示資料來源。

### 與其他 QA 協調

09_i 是跨場版綜合 QA，與下列場內 QA 互補：

| 場內 QA | 09_i 對應面向 |
|---|---|
| 09_b 角色聲線一致性 | 09_i 聲線漂移是 09_b 的跨場擴展；09_b 看單場內部一致，09_i 看多場變化 |
| 09_d 資訊控制 | 09_i 資訊洩漏是 09_d 的跨場擴展；09_d 看單場是否提前，09_i 看累積揭露邏輯 |
| 09_g 節奏感 | 09_i 節奏 arc 使用 09_g 的場級指標 |
| 09_h 對話張力 | 09_i 張力走勢使用 09_h 的場級指標 |

若 09_b / 09_d 場內步驟已偵測到跨場問題，09_i 跨場視角需再驗證一次；同時命中時，以 09_i 作為跨場層級結論來源。

## frontmatter M7/M10 對齊

本 Template 檔本身維持空白 Template frontmatter：

```yaml
qa_type: CROSS_SCENE_CONTINUITY
entities: []
depends_on: []
weight: {}
```

Instance 執行 `/qa` 產出實際 QA 報告時，必須填入下游欄位：

```yaml
狀態：DRAFT
版本：v0.1
最後更新：YYYY-MM-DD
適用範圍：CH<n>_S<m>_<簡稱>_dialogue_vXX 跨場一致性檢查報告（scope=<...>）
優先級：中

---
entities: [S-<n>-<m>, <跨場範圍涵蓋的其他 S-*>]
depends_on: [<本場相關 C-*>, <本場相關 R-*-*>, <跨場範圍涵蓋的所有 C-*>]
weight: {S-<n>-<m>: 0.125}
scene_id: S-<n>-<m>
source_task: 07_scene_tasks/CH<n>_S<m>_台詞任務包.md
source_dialogue: <被檢查的台詞檔路徑>
source_dialogues: null
pipeline_state: <被檢查台詞檔的 pipeline_state>
mode_tag: <被檢查台詞檔的 mode_tag>
qa_decision: PASS | FAIL | NEEDS_REVIEW
qa_type: CROSS_SCENE_CONTINUITY
cross_scene_scope: chapter | arc | all
---
```

`cross_scene_scope` 是 09_i 專屬擴充欄位。若資料格式支援狀態尚未落地，實際報告需標 `[NEEDS_SCHEMA_SUPPORT]` 並避免改動 parser。

## 通用化原則

- 不含任何作品專屬角色、世界觀、主線、專有名詞或場景細節。
- 本模板只定義通用檢查方法；Instance 跑 `/qa` 時才填入該作品的 entity list、dependency 與 weight。
- Template frontmatter 固定使用 `entities: []`、`depends_on: []`、`weight: {}`。
- 本模板不新增 `00_p` 實檔；僅保留與 QA 擴充協議接點 schema 的對齊聲明。
- 本模板不新增獨立 continuity entity；跨場一致性由 `qa_type: CROSS_SCENE_CONTINUITY` 報告承載。
- 若未來 user-defined 09_x 模板擴充，應透過 `qa_type_registry` 的 `user_extensions` 機制處理，不修改本 core 模板。
