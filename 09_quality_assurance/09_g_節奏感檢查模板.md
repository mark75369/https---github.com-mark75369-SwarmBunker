# 09_g 節奏感檢查模板（通用版）

## 用途

本模板用於檢查單場台詞檔的節奏感。

節奏感不是單純檢查句子長短，而是檢查一場戲在句長分布、句長變異度、長短句交替、段落呼吸感與章節節奏定位之間是否一致。

本模板是通用 Template，不含任何作品專屬內容。Instance 執行 `/qa` 時，才依該作品的任務包、角色聲線卡、章節節奏與台詞檔填入實際檢查對象。

## qa_type

本模板對應 `qa_type_registry` 的 enum 值：

```yaml
qa_type: RHYTHM
```

`RHYTHM` 是 8 份 `/qa` 必跑報告之一。依 SPEC §12.7 的序列，09_g 在 09_b 角色聲線一致性之後印出，因為不同角色的典型句長與停頓習慣會影響節奏判定。

## 5 層檢查框架

### 1. 檢查對象定位

- 讀取目標台詞檔。
- 從 `source_task` 反查任務包。
- 讀取任務包中的章節節奏、風格要求、場景壓力與出場角色。
- 確認本模板只檢查單場節奏；跨場節奏 arc 交由 09_i。

### 2. 台詞句段切分

- 排除心理 OS 與場景指示。
- 以「。！？」與換行做初步切句。
- 標點不計入句長；中文一字一單位。
- 依角色發言與換行密度切成段落。

### 3. 量化指標計算

- 統計句數、平均句長、標準差、變異係數、最長句與最短句。
- 計算相鄰句長變化形成的 `swing_score`。
- 統計短句、中句、長句比例。
- 統計段落內句數、段平均句長與停頓標示。

### 4. 與上游基準對齊

- 對照任務包章節節奏：high / mid / low。
- 對照角色聲線卡的典型句長與停頓習慣。
- 對照 09_h 對話張力的張力需求。
- 若節奏問題反映高層類型偏移，交叉通知 09_f。

### 5. 分級與輸出

- `PASS`：節奏統計落在任務包與角色聲線允許範圍內。
- `NEEDS_REVIEW`：節奏異常但可能是場景設計或角色特殊狀態造成。
- `FAIL`：節奏顯著破壞場景可讀性、角色聲線或章節節奏定位。

## 檢查 algorithm

09_g 是機械統計層 QA，與 09_b 聲線一致性（語感層）互補。algorithm 跑下列步驟。

### 步驟 1：句長分布統計

對目標台詞檔（一場戲），對每句台詞（不含心理 OS / 場景指示）：

1. 句長定義：字元數（中文一字一單位；標點不計）；切句以「。！？\n」為界。
2. 對全場累積一份 list `[L1, L2, L3, ...]`。
3. 計算：
   - 句數 `N`
   - 平均句長 `μ`
   - 標準差 `σ`
   - 變異係數 `CV = σ / μ`
   - 最長句 `Lmax` / 最短句 `Lmin`

同時統計句長分布：

| 分類 | 定義 | 報告用途 |
|---|---|---|
| 短句 | < 10 字 | 檢查是否過度碎裂或高張力短打 |
| 中句 | 10-25 字 | 檢查是否為主要穩定區 |
| 長句 | > 25 字 | 檢查是否有冗長、說明腔或節奏拖慢 |

### 步驟 2：句長變異度判定

| 指標 | 健康範圍 | 越界判定 |
|---|---|---|
| `CV` | 0.4-1.2 | < 0.4：句長過於均勻，可能有機械感；> 1.2：句長劇烈震盪，可能風格失控 |
| `μ` | 8-22 字 | < 8：過於碎裂；> 22：過於冗長 |
| `Lmax` | ≤ 60 字 | > 60：超長句警示，不必然 FAIL，但需標 NEEDS_REVIEW |

判定時不得只看全場平均值。若某角色的聲線卡明示短句或長句偏好，需以角色聲線卡做加權說明。

### 步驟 3：長短句交替指標

定義 `swing_score`：相鄰兩句長度差 / 平均句長，全場求平均。

| `swing_score` | 判定 |
|---|---|
| < 0.3 | 節奏單一，連串長句或連串短句；通常 FAIL |
| 0.3-1.0 | 健康 |
| > 1.5 | 節奏過於跳躍，讀感破碎；標 NEEDS_REVIEW |

額外檢查：

- 連續短句段落是否符合高張力或打斷需求。
- 連續長句段落是否是必要說明，或只是角色替劇情整理。
- 長句後是否有短句、停頓或動作切分提供呼吸點。
- 短句連發是否仍有攻防功能，而不是單純把句子切碎。

### 步驟 4：段落呼吸感

把場景按角色換行與發言間隔切「段」。對每段：

1. 段內句數 `n_p`
2. 段平均句長 `μ_p`
3. 段是否含停頓、沉默、省略號、短動作或可讀的留白

判定：

- 全場若無任何停頓 / 沉默 / 省略號段且 `μ > 18`，標「對話無呼吸間隙，推進過密」（NEEDS_REVIEW）。
- 若段內 `n_p > 8` 且 `μ_p > 15`，該段標「對話冗長，缺乏節奏切換」（FAIL）。
- 若每段都很短但缺乏角色意圖推進，標「碎裂但無攻防功能」（NEEDS_REVIEW）。
- 若段落留白只靠空行堆砌，沒有台詞功能或動作功能，標「形式留白不足以支撐節奏」（NEEDS_REVIEW）。

### 步驟 5：與任務包「章節節奏」對齊

讀任務包 `章節節奏` 欄（high / mid / low）：

| 章節節奏 | 預期 `μ` | 預期 `CV` |
|---|---|---|
| high（衝突場 / 高潮） | 6-14 字 | 0.5-1.2 |
| mid（推進場 / 過場） | 10-20 字 | 0.4-0.9 |
| low（回憶 / 內省 / 抒情） | 14-30 字 | 0.4-0.7 |

實際統計值若超出預期範圍，標「節奏與章節定位不符」（NEEDS_REVIEW），並附建議方向。

### 步驟 6：與其他 QA 協調

| 對 | 09_g 的處理 |
|---|---|
| 09_a AI 味 | AI 味的結構模板化常伴隨節奏單一；09_g 提供量化證據 |
| 09_b 聲線 | 不同角色的典型句長不同；09_g 句長判定按出場角色加權，不用單一全場閾值套所有角色 |
| 09_f 類型偏移 | 高層偏移常伴隨節奏失控；09_g 是症狀偵測，09_f 是病因判定 |
| 09_h 對話張力 | 張力高低會反映節奏；09_g 提供 `swing_score`，09_h 可用此推導張力強度 |
| 09_i 跨場一致性 | 09_i 用 09_g 的場級指標做跨場 arc 比對 |

## 通用化原則

- 不含任何作品專屬角色、世界觀、主線、專有名詞或場景細節。
- 本模板只定義通用檢查方法；Instance 跑 `/qa` 時才填入該作品的 entity list、dependency 與 weight。
- Template frontmatter 固定使用 `entities: []`、`depends_on: []`、`weight: {}`。
- 本模板不新增 `00_p` 實檔；僅保留與 QA 擴充協議接點 schema 的對齊聲明。
- 若未來 user-defined 09_x 模板擴充，應透過 `qa_type_registry` 的 `user_extensions` 機制處理，不修改本 core 模板。
