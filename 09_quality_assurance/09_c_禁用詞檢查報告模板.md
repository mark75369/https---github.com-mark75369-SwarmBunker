# 1. 文件目的

本文件用於建立禁用詞與慎用詞的 QA 檢查流程。

禁用詞檢查不是單純找敏感詞，而是檢查「這個詞在這個時間點、這個角色、這個場景、這段關係中能不能出現」。

本文件檢查範圍包括：

- 全作品禁用詞
- 全作品慎用詞
- AI 高頻詞與套語
- 未解禁專有名詞
- 未解禁俗稱、真名、暱稱、血緣稱呼
- 角色個人禁用詞
- 關係禁語
- 弧線提前成長詞
- 資訊劇透詞
- 不符合角色階級、陣營、職業的詞
- 不符合世界語言規格的現代詞、網路詞、外來詞
- 玩家選項中不該提前知道的詞

本文件的目的，是讓台詞在進入 FINAL 前，能逐項確認詞彙風險。

# 本文件專屬檢查維度

> 速查卡（v0.2 新增）：本文件在禁用詞 QA 流程中需檢查的具體維度。詳細規則仍以後續各章為準。

| 維度 | 說明 |
|---|---|
| 禁用詞 | 全作品 / 本場 / 角色 / 關係層級的禁用詞 |
| 慎用詞 | 高頻 AI 詞、抽象詞、套語、可重複的高風險詞 |
| 替代表達 | 是否提供可用替代詞 / 暗示 / 行動 / 物件 |
| 世界觀不相容詞 | 不符合時代、科技、宗教、文化語境的詞 |
| 現代詞污染 | 現代詞、網路詞、外來語、翻譯腔 |

詞彙風險的核心問題不是「能不能用」，而是：「這個詞現在能不能由這個角色說出來？」

# 3. 禁用詞檢查原則

## 3.1 禁用詞不只是一張詞表

同一個詞可能在不同情境下有不同狀態。

例如：

| 情境 | 詞彙狀態 |
|---|---|
| 前期主線 | 禁用 |
| 中期暗示 | 慎用 |
| 後期揭露後 | 可用 |
| 特定角色台詞 | 可用 |
| 玩家選項 | 禁用 |
| UI 文本 | 可用 |
| NPC 閒聊 | 禁用 |

因此檢查時不能只看詞本身，還要看：

- 誰說
- 對誰說
- 何時說
- 在什麼場合說
- 玩家此時知不知道
- 角色此時知不知道
- 該詞是否已在本場解禁

---

## 3.2 慎用詞不是絕對禁止

慎用詞可以在符合條件時使用，但必須有理由。

慎用詞常見原因：

- 太 AI 味
- 太現代
- 太抽象
- 太主題化
- 太像心理總結
- 太像任務提示
- 太容易劇透
- 太不符合角色階級
- 太容易被重複濫用

慎用詞若使用，應記錄使用原因與是否替代過。

---

## 3.3 禁用詞檢查應保護角色與資訊節奏

禁用詞檢查的核心不是讓文字更保守，而是保護：

- 角色聲線
- 關係進度
- 資訊揭露
- 世界語感
- 台詞自然度
- 玩家體驗
- 商業級文本穩定性

若一個詞「正確但不該出現」，仍應判定為問題。

---

## 3.4 區分自動刪除與人類裁決

禁用詞檢查應區分：

- **自動刪除**：全作品禁用詞、劇透詞、無條件違規詞。
- **人類裁決**：若某詞違反通用規則但符合角色聲線、作品風格或資訊設計，應標記為人類裁決，不得自動刪除。

若難以判定，應標記為「需人類確認」。

# 破格版本處理規則（REVIEW_REQUIRED）

> v0.2 新增。**破格句不是違規豁免，而是機械刪除豁免。**

Mode Tag 為 `EXPERIMENTAL`（`_破格` 或 `_探索` 來源）的版本，本文件 QA 不得自動 `PASS`，也不得自動 `FAIL`，必須一律標記為：

```text
REVIEW_REQUIRED 或 HUMAN_ARBITRATION_REQUIRED
```

QA 報告中必須列出五欄：

| 破格內容 | 命中的風險維度 | 可保留理由 | 修正建議 | 需人類裁決原因 |
|---|---|---|---|---|
|  | 全作品禁用 / 本場禁用 / 未解禁 / 角色錯詞 / 階級錯詞 / 劇透詞 / 主題宣言 / 現代違和 / 其他 |  |  |  |
|  |  |  |  |  |

禁用詞專屬補充：

- 禁用詞破格可能有風格價值（角色獨特語感），也可能破壞世界觀、分級規範或語言系統。**不得單看通用規則判定。**
- 若破格詞屬於 §4.1「全作品禁用」(`FORBIDDEN`)、「劇透詞」、「未解禁名詞 / 稱呼」(`LOCKED`)，傾向 `HUMAN_ARBITRATION_REQUIRED`，不得 `REVIEW_REQUIRED` 後自動放行。
- 若破格詞屬於「慎用詞」(`CAUTION`)、「角色錯詞」、「階級錯詞」，傾向 `REVIEW_REQUIRED`。
- 本表內容須在進入定稿前整理至 09_e §8.3 的人類裁決紀錄表。

§3.4 既有「區分自動刪除與人類裁決」、§4.1 詞彙狀態 enum、§14「禁用詞替換建議」等仍適用，作為更細的處理指引；本節為跨模板的統一判定骨架。

# 4. 詞彙風險分類

## 4.1 詞彙狀態

| 狀態 | 意義 | 處理方式 |
|---|---|---|
| ALLOWED | 可用 | 無需修改 |
| CAUTION | 慎用 | 需確認使用理由 |
| FORBIDDEN | 禁用 | 必須移除或替換 |
| LOCKED | 尚未解禁 | 必須等到指定場景後才可用 |
| ROLE_ONLY | 僅特定角色可用 | 檢查說話者 |
| CONTEXT_ONLY | 僅特定情境可用 | 檢查場景與語氣 |
| UI_ONLY | 只允許 UI / 任務文本使用 | 不應出現在角色台詞 |
| DIALOGUE_ONLY | 只允許角色台詞使用 | 不應出現在 UI / 任務文本 |
| DEPRECATED | 已棄用 | 不得作為正式用詞 |

---

## 4.2 問題類型

| 類型 | 說明 |
|---|---|
| 全作品禁用 | 在任何正式文本中不得使用 |
| 本場禁用 | 本場不允許出現 |
| 未解禁名詞 | 劇情尚未允許出現的正式名詞 |
| 未解禁稱呼 | 真名、暱稱、血緣稱呼、關係稱呼尚未解禁 |
| AI 高頻詞 | 容易產生 AI 味的高頻抽象詞或套語 |
| 角色錯詞 | 不符合角色身份、聲線或知識權限 |
| 階級錯詞 | 不符合階級、地域、職業或陣營語言 |
| 劇透詞 | 詞本身會暴露後期真相 |
| 主題宣言詞 | 讓台詞變成作者論述 |
| 現代違和詞 | 不符合世界觀語感的現代詞、網路詞、外來詞 |
| 格式錯詞 | 不符合 UI、任務、台詞格式規範 |

2. 不得只用字面搜尋，必須檢查語境、角色、關係與解禁時間。

# 18. 最終原則

禁用詞檢查的目標，不是讓台詞變得安全但無味，而是確保每個詞都出現在正確的人、正確的時間、正確的關係與正確的世界語感裡。

一個詞可以本身沒有問題，卻在錯誤場景中破壞：

- 角色聲線
- 關係進度
- 資訊節奏
- 玩家懸念
- 世界語言
- 台詞自然度

因此禁用詞 QA 的核心問題不是「這個詞能不能用」，而是：

> 這個詞現在能不能由這個角色說出來？

若答案不明確，就不能直接放進 FINAL。
