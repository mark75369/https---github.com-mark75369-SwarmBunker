# 09_h 對話張力檢查模板（通用版）

## 用途

本模板用於檢查單場台詞檔的對話張力。

對話張力不是單純看角色是否爭吵，而是檢查每句台詞是否在推進、讓步、揭穿、反擊之間形成有效攻防，並與任務包中的角色目標、章節節奏、風格要求與高風險場景類型對齊。

本模板是通用 Template，不含任何作品專屬內容。Instance 執行 `/qa` 時，才依該作品的任務包、角色聲線卡、關係矩陣與台詞檔填入實際檢查對象。

## qa_type

本模板對應 `qa_type_registry` 的 enum 值：

```yaml
qa_type: DRAMATIC_TENSION
```

`DRAMATIC_TENSION` 是 8 份 `/qa` 必跑報告之一。依 SPEC §12.7 的序列，09_h 在 09_d 資訊控制之後、09_b 角色聲線一致性之前印出，因為張力強度需要先避開資訊提前揭露，再影響聲線與句長判定。

## 5 層檢查框架

### 1. 檢查對象定位

- 讀取目標台詞檔。
- 從 `source_task` 反查任務包。
- 讀取角色表層目標、角色真實目標、章節節奏、風格要求與 risk_type。
- 確認本模板只檢查單場攻防張力；跨場張力 arc 交由 09_i。

### 2. 台詞功能標記

- 排除心理 OS 與場景指示。
- 對每句台詞標記 PUSH / YIELD / EXPOSE / COUNTER 四選一。
- 每句再標 1-5 力度分。
- 若一句同時具備多個動作，以對場景狀態改變最大的動作為主標籤，次要動作寫入備註。

### 3. 頻率與力度計算

- 計算 `PUSH_pct`、`YIELD_pct`、`EXPOSE_pct`、`COUNTER_pct`。
- 計算 `tension_total` 與 `softness_total`。
- 計算全場平均攻防力度 `μ_intensity`。
- 標出張力過低、張力過高或張力方向錯位的段落。

### 4. 與上游基準對齊

- 對照角色表層目標與真實目標。
- 對照章節節奏 high / mid / low。
- 對照風格要求中的克制、尖銳、壓抑、失控等限制。
- 對照 00_b §6 高風險場景處理方式。

### 5. 分級與輸出

- `PASS`：攻防頻率與力度支撐場景目標。
- `NEEDS_REVIEW`：張力 pattern 偏離，但可能是反向設計或角色策略。
- `FAIL`：張力不足、張力過剩、資訊揭穿超前，或攻防方向違反任務包核心目標。

## 檢查 algorithm

09_h 把對話張力量化為四種「動作」的頻率與力度。algorithm 跑下列步驟。

### 步驟 1：對每句台詞標四選一動作標籤

| 標籤 | 定義 | 典型語言訊號 |
|---|---|---|
| `PUSH`（推進） | 說話者試圖讓對方接受某事 / 做某事 / 承認某事 | 命令式、要求式、直問式、結論式句型 |
| `YIELD`（讓步） | 說話者後退 / 接受 / 認輸 / 改變立場 | 妥協詞、軟化詞、結束話題 |
| `EXPOSE`（揭穿） | 說話者攤牌 / 揭穿對方 / 拆穿謊言 / 點破暗示 | 直陳事實、反問、揭露邏輯漏洞 |
| `COUNTER`（反擊） | 說話者反過來壓制 / 質疑 / 翻轉局勢 | 反問、嘲諷、反向邏輯、提出對手弱點 |

agent 對全場逐句標籤。含心理 OS 的句不算，因為 OS 屬內心，不算對話動作。

標記規則：

- 只標實際改變對話狀態的功能，不標情緒形容。
- 若一句只是補充設定但不改變攻防，標為低力度 PUSH 或列入「無有效攻防」備註。
- 若角色沉默、轉移話題或故意不答，視語境可標 YIELD 或 COUNTER，但必須寫明理由。
- 若 EXPOSE 涉及資訊揭露，必須同步交叉 09_d。

### 步驟 2：統計四種動作頻率

對全場：

| 指標 | 計算 |
|---|---|
| `PUSH_pct` | PUSH 數 / 總句數 |
| `YIELD_pct` | YIELD 數 / 總句數 |
| `EXPOSE_pct` | EXPOSE 數 / 總句數 |
| `COUNTER_pct` | COUNTER 數 / 總句數 |
| `tension_total` | (PUSH + EXPOSE + COUNTER) / 總句數，即進攻力總分 |
| `softness_total` | YIELD / 總句數，即讓步力總分 |

判定輔助：

- 高張力場若 `tension_total` 偏低，通常代表衝突不足。
- 低張力場若 `tension_total` 偏高，通常代表場景過度尖銳。
- 若 `YIELD_pct` 過高但任務包不是退讓或告別場，可能代表角色過早放棄攻防。
- 若 `EXPOSE_pct` 過高，必須確認資訊揭露時點。

### 步驟 3：攻防力度量化（per 句）

對每句標籤後再給 1-5 力度分：

| 力度 | PUSH 範例方向 | EXPOSE 範例方向 | COUNTER 範例方向 | YIELD 範例方向 |
|---|---|---|---|---|
| 1 | 隨口提一句要求 | 隱晦點出 | 微弱反問 | 表面附和 |
| 3 | 明確要求 | 直接點破 | 中度反擊 | 半推半就接受 |
| 5 | 強硬命令且不容拒絕 | 全面攤牌 | 翻盤性反擊 | 徹底投降 |

對全場求平均 `μ_intensity`。

力度判定必須依任務包與角色聲線卡校正：

- 某角色慣常克制時，力度 3 可能已經是高張力。
- 某角色慣常攻擊時，力度 3 可能只是基本表達。
- 不得把吵鬧等同於高張力；張力必須能改變場景狀態。
- 不得把沉默等同於低張力；沉默若壓迫對方，也可能是 COUNTER。

### 步驟 4：與任務包目標對齊

讀任務包：

| 欄位 | 對齊規則 |
|---|---|
| `角色表層目標` / `角色真實目標` | 主角的 PUSH 方向應與真實目標一致；若 PUSH 朝表層目標但 EXPOSE 揭穿表層，場景結構可視為合理 |
| `章節節奏` high | `tension_total` 應 ≥ 0.5；否則標「衝突場張力不足」（FAIL） |
| `章節節奏` low | `tension_total` 應 ≤ 0.3；否則標「低張力場攻防過高」（FAIL） |
| `風格要求` 克制 | `μ_intensity` ≤ 3；否則標「力度過剩」（NEEDS_REVIEW） |

若任務包目標缺漏，09_h 不擅自補寫目標，只在報告中標「上游缺漏影響張力判定」。

### 步驟 5：高風險場景額外檢查

若任務包 `risk_type` 命中 00_b §6 高風險場景類型：

| risk_type | 預期張力 pattern |
|---|---|
| 告別 | YIELD 高 + EXPOSE 中 + COUNTER 低 |
| 告白 | PUSH 中 + EXPOSE 中 + 力度 3-5 |
| 揭穿 | EXPOSE 主導（≥ 0.4） |
| 背叛 | COUNTER 高 + EXPOSE 高 + YIELD 低；被背叛方例外需另註 |

不符 pattern 時標 NEEDS_REVIEW，不必然 FAIL。若任務包明確要求反向 pattern，報告須註明人類可裁決保留；若任務包沒有交代，則要求補充或修正。

### 步驟 6：與其他 QA 協調

| 對 | 協調規則 |
|---|---|
| 09_g 節奏感 | 高張力場常伴短句節奏；09_h 引用 09_g 的 `swing_score` 做交叉驗證 |
| 09_b 聲線 | 角色聲線卡可能有典型攻防偏好；09_h 按此加權判定角色是否 OOC |
| 09_d 資訊控制 | EXPOSE 動作必須符合 05_d 可揭露資訊；揭露超前由 09_d 抓 |
| 09_f 類型偏移 | PUSH / COUNTER 密度與情緒力度若不符類型基準，交叉通知 09_f |
| 09_i 跨場一致性 | 09_i 可使用 09_h 的場級張力指標做跨場 arc 比對 |

## 通用化原則

- 不含任何作品專屬角色、世界觀、主線、專有名詞或場景細節。
- 本模板只定義通用檢查方法；Instance 跑 `/qa` 時才填入該作品的 entity list、dependency 與 weight。
- Template frontmatter 固定使用 `entities: []`、`depends_on: []`、`weight: {}`。
- 本模板不新增 `00_p` 實檔；僅保留與 QA 擴充協議接點 schema 的對齊聲明。
- 若未來 user-defined 09_x 模板擴充，應透過 `qa_type_registry` 的 `user_extensions` 機制處理，不修改本 core 模板。
