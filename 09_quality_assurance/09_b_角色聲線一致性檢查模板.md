# 1. 文件目的

本文件用於檢查台詞是否符合角色聲線。

角色聲線一致性不是檢查台詞是否好聽，而是檢查：

- 這句話是否像這個角色會說的話？
- 這個角色此時是否能說出這種話？
- 這個角色是否使用了不屬於自己的詞彙？
- 這個角色是否突然變得過度成熟、過度溫柔、過度理性或過度文藝？
- 這個角色面對不同對象時，語氣是否有正確變化？
- 這個角色在當前弧線階段是否提前成長？
- 這個角色是否忘記自己的階級、職業、陣營、創傷與遮掩方式？
- 拿掉角色名後，玩家是否仍能辨識這是誰？

本文件與 `09_quality_assurance/09_a_ai味qa報告模板.md` 不同。
`09A` 檢查通用 AI 味；本文件專門檢查角色專屬聲線。

# 3. 聲線一致性判定原則

## 3.1 角色聲線高於漂亮台詞

若一句台詞文筆漂亮，但不符合角色聲線，應判定為失敗。

角色聲線優先於：

- 文學感
- 流暢度
- 情緒完整度
- 主題表達
- 玩家理解便利
- AI 生成的漂亮句

一個角色不一定會用最清楚、最成熟、最有詩意的方式說話。

---

## 3.2 聲線一致不等於永遠一樣

角色面對不同人、不同壓力、不同關係階段時，語氣可以改變。

但變化必須有依據：

- 場景壓力
- 角色弧線階段
- 關係變化
- 權力位置
- 情緒失控
- 身份偽裝
- 資訊揭露
- 故意說謊或表演

沒有依據的變化，應視為聲線漂移。

---

## 3.3 角色聲線包含「不會說什麼」

檢查聲線時，不只看角色說了什麼，也要看他不該說什麼。

需要檢查：

- 是否說出他此時不該承認的情緒？
- 是否使用他不會使用的稱呼？
- 是否說出他不會知道的資訊？
- 是否用不屬於他階級、職業或地域的詞？
- 是否提前完成弧線？
- 是否替作者說主題？

角色的禁語與迴避方式，是聲線的一部分。

---

## 3.4 QA 區分錯誤與風格風險

QA 檢查聲線時，需區分：

- **錯誤**：明顯違反角色聲線卡、關係狀態或弧線階段。
- **有風格價值的風險**：違反通用規則，但符合角色專屬風格、作品美學或資訊設計。

若某句違反通用規則但符合角色、作品風格或資訊設計，應標記為人類裁決，不得自動刪除。

# 破格版本處理規則（REVIEW_REQUIRED）

> v0.2 新增。**破格句不是違規豁免，而是機械刪除豁免。**

Mode Tag 為 `EXPERIMENTAL`（`_破格` 或 `_探索` 來源）的版本，本文件 QA 不得自動 `PASS`，也不得自動 `FAIL`，必須一律標記為：

```text
REVIEW_REQUIRED 或 HUMAN_ARBITRATION_REQUIRED
```

QA 報告中必須列出五欄：

| 破格內容 | 命中的風險維度 | 可保留理由 | 修正建議 | 需人類裁決原因 |
|---|---|---|---|---|
|  | 聲線漂移 / 提前成長 / 階級錯詞 / 關係過早 / 其他 |  |  |  |
|  |  |  |  |  |

聲線專屬補充：

- 聲線破格可能是角色情緒爆點（值得保留），也可能是角色崩壞（必須修正）；不得單看通用規則判定。
- 若破格句符合角色弧線階段或場景壓力，但違反聲線卡細節，傾向 `REVIEW_REQUIRED`。
- 若破格句明顯違反角色階級、職業、知識權限，傾向 `HUMAN_ARBITRATION_REQUIRED`。
- 本表內容須在進入定稿前整理至 09_e §8.3 的人類裁決紀錄表。

§3.4 既有「QA 區分錯誤與風格風險」、§6.2「可保留句」、§17「聲線修正建議」等仍適用，作為更細的處理指引；本節為跨模板的統一判定骨架。

# 19. Codex 使用規則

> **註（2026-05-08，v0.2）**：本節原名「Codex 使用規則」。目前 repo 落地與檔案修改端已改為 **Claude**；條文中的「Codex」可視為「Repo Execution 端」之歷史代稱。新增的 Claude 權限規則請見後續「§ Repo Execution / Claude 權限」一節。

當 Codex 使用本文件檢查台詞時，應遵守：

1. 不得只說「聲線不太像」，必須指出具體行號與原因。
2. 必須對照角色聲線卡、關係矩陣與弧線階段。
3. 不得為了讓聲線更像而擅自改變資訊揭露或關係狀態。
4. 不得把所有角色都改成更尖銳或更口語，必須依角色聲線處理。
5. 若缺少角色聲線卡，應標註缺漏，不得假裝已通過。
6. 若聲線問題來自任務包不足，應指出上游缺漏。
7. 若台詞有可保留句，應標記。
8. 若需重寫，應說明是聲線、弧線、關係還是資訊權限問題。
9. 檢查後不得擅自標記 FINAL。

# Repo Execution / Claude 權限

> v0.2 新增。repo 落地端已從 Codex 移到 Claude。本節用於明確 Claude 在聲線 QA 流程中的權限分層。詳細執行規則沿用 §19「Codex 使用規則」（歷史代稱），本節為其在 Claude 實作端的權限補充。

## Claude 可寫

- 09_b 聲線 QA 報告草稿（含逐句問題標記、可保留句、需重寫句）
- 09_e 中對應的人類裁決草稿

## Claude 可建議但不可自行決定

- QA Decision 變更（`PASS` / `FAIL` / `REVIEW_REQUIRED` / `HUMAN_ARBITRATION_REQUIRED`）
- 是否升 Lifecycle Status（`REVIEW_CANDIDATE` 等）
- 破格聲線句是否最終保留
- 角色聲線卡是否需要更新

## Claude 不可自行做

- 把 `REVIEW` 升 `FINAL`
- 把 `FINAL` 升 `LOCKED`
- 把破格聲線句直接刪除
- 把台詞內容直接回寫角色聲線卡 / Bible
- 修改已確認 `FINAL` / `LOCKED` 紀錄
- 推翻人類已下達的聲線裁決

# 21. 最終原則

角色聲線一致性 QA 的目標，不是讓角色永遠重複同一種說話方式，而是讓角色在變化時仍然像自己。

好的聲線檢查應該能判斷：

- 這句話是否符合角色本人的語言習慣
- 這句話是否符合他面對此人的關係狀態
- 這句話是否符合他目前的弧線階段
- 這句話是否符合他所處的階級、陣營、職業與世界壓力
- 這句話是否說出了他此時不該說出口的真心

角色可以變，但不能無理由漂移。  
角色可以成長，但不能提早成長。  
角色可以說漂亮話，但前提是那漂亮話真的屬於他。
