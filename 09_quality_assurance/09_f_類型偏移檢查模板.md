# 09_f 類型偏移檢查模板（通用版）

## 1. 文件目的

本模板用於檢查台詞是否把作品拖向類型偏移。

本檢查以作品專屬 `00_b` 的 §1、§2、§3、§4、§6 為基準，判定單場台詞是否在基調、死亡與髒話處理、規模定位、類型偏移風險、高風險場景處理方式上偏離。

本模板是 `/qa` 五份報告之一，閱讀順序最優先。若本報告判定重大偏移，後續 09_d、09_b、09_a、09_c 的判定基準都可能需要重看。

## 3. 類型偏移判定原則

### 3.1 類型偏移不等於一般風格越界

某段台詞風格特殊，不必然是類型偏移。

判定時先問：

1. 它是否命中 `00_b §4` 的禁忌方向。
2. 它是否破壞 `00_b §1` 的基調定位。
3. 它是否把場景規模推向 `00_b §3` 不允許的尺度。
4. 它是否用不符合 `00_b §6` 的方式處理高風險場景。

若命中上述任一項，即使句子本身寫得順，也要標偏移。

### 3.2 高層偏移與局部偏移

| 類型 | 定義 | 處理 |
|---|---|---|
| 高層偏移 | 整場戲方向偏離作品基調或規模定位 | 通常 FAIL |
| 局部偏移 | 個別句子、個別角色短暫偏離 | 視密度判定 FAIL 或 NEEDS_REVIEW |
| 邊緣案例 | 偏移可能是有意設計，但任務包沒有交代 | 標 NEEDS_REVIEW |

### 3.3 不替代其他 QA

09_f 不做下列工作：

- 不逐字做禁用詞機械比對；那是 09_c。
- 不判定角色知道什麼；那是 09_d。
- 不完整判定角色聲線；那是 09_b。
- 不完整判定 AI 味；那是 09_a。

09_f 只判斷「這場台詞是否把作品拉向不該去的類型方向」。

每個問題必須指出：

- 命中的 `00_b` 來源 section
- 命中位置
- 為什麼構成類型偏移
- 建議修法

## 5. 對照 00_b §1 類型語氣定位

### 5.1 基調符合度

檢查步驟：

1. 讀 `00_b §1` 的 1-3 個基調詞。
2. 逐句標示：符合 / 中性 / 違反。
3. 若違反句集中出現在同一角色，交叉通知 09_b。
4. 若違反句改變整場戲方向，標高層偏移。

### 5.2 敘事氣質一致性

檢查整場戲是否與 `00_b §1` 的「敘事氣質一句話」一致。

要注意：

- 整場戲是否忽然變成另一種作品類型。
- 角色是否開始替作品說明主題。
- 情緒處理是否變成與作品基調相反的宣言。
- 場景收束是否變得過於整齊、過於安慰或過於說教。

### 5.3 判定格式

| 句子 / 段落 | 00_b §1 基準 | 判定 | 理由 | 建議 |
|---|---|---|---|---|
|  |  | 符合 / 中性 / 違反 |  |  |

## 6. 對照 00_b §2 髒話與死亡處理偏好

### 6.1 髒話強度檢查

檢查：

- 髒話頻率是否符合 `00_b §2`。
- 髒話強度是否符合角色與場景。
- 是否用髒話替代具體行動或潛台詞。
- 是否因髒話過度而改變作品類型氣質。

判定：

| 情況 | 判定 |
|---|---|
| 髒話頻率與 00_b 一致 | PASS |
| 髒話偏多但不改變場景方向 | NEEDS_REVIEW |
| 髒話把場景推向不符基調的表演性粗俗 | FAIL |

### 6.2 死亡 / 告別場景處理方式檢查

若本場涉及死亡、告別、犧牲、失去或重大不可逆事件，必須對照 `00_b §2`。

檢查：

- 是否符合死亡處理偏好。
- 是否出現過度總結、過度告白、過度安慰。
- 是否把原本應該以行動或殘留物呈現的事件改成長篇說明。
- 是否把原本應該有明確情緒的場景寫得過度機械。

## 7. 對照 00_b §3 規模定位

### 7.1 規模賭注一致性

檢查本場台詞暗示的賭注規模，是否符合 `00_b §3`。

| 00_b §3 規模 | 檢查重點 |
|---|---|
| 對角色要命 | 不應無故升成全局危機 |
| 對特定群體要命 | 不應忽然變成單一角色小情緒，也不應無故升成全局危機 |
| 對世界要命 | 不應把重大危機寫得像私人小摩擦，除非任務包明示反差 |
| 混合 / 其他 | 依 00_b 的排序與說明判定 |

### 7.2 規模上綱警示

若 `00_b §3` 限定為較小尺度，而台詞中出現過大的責任、使命、群體命運或終局宣告，通常標 FAIL。

若任務包明示角色在誇張、誤導或演戲，標 NEEDS_REVIEW，並要求人類確認是否保留。

## 8. 對照 00_b §4 類型偏移風險清單

### 8.1 逐條對照

`00_b §4` 是本報告的最高權重來源之一。

檢查流程：

1. 逐條讀取 §4 禁止方向。
2. 對照台詞與場景收束。
3. 若命中任一條，列為 FAIL，除非任務包明確要求角色故意做出錯誤表演。

### 8.2 命中位置標記

報告必須標出命中位置：

| 00_b §4 條目 | 命中句 / 段 | 偏移等級 | 理由 | 建議 |
|---|---|---|---|---|
|  |  | 局部 / 高層 / 嚴重 |  |  |

不得只寫「有偏移」。

## 9. 對照 00_b §6 高風險場景處理方式

### 9.1 識別場景類型

agent 必須從三個來源確認場景類型：

1. 任務包「本場戲劇目的」。
2. 任務包「角色目標」。
3. `06_a` 該場 risk_type 註記。

若三者不一致，標 NEEDS_REVIEW。

### 9.2 處理方式符合度

若本場是高風險場景，必須對照 `00_b §6` 對應子節。

檢查：

- 是否符合該類型的正確處理方式。
- 是否使用 `00_b §6` 明確避免的處理方式。
- 是否讓高風險場景變成主題宣言、劇情整理或心理總結。
- 是否把應該壓住的資訊說破。

### 9.3 高風險場景判定表

| 場景類型 | 00_b §6 規則 | 台詞處理 | 判定 | 建議 |
|---|---|---|---|---|
|  |  |  | PASS / FAIL / NEEDS_REVIEW |  |

## 10. 角色 vs 類型氣質檢查

### 10.1 讀角色聲線卡

對每個出場角色，讀其聲線卡的「與類型氣質合規」段。

檢查：

- 該角色是否已標「容易拖向 X 偏移」。
- 本場是否真的出現 X 偏移。
- 該偏移是角色本身設計的一部分，還是寫作失控。

### 10.2 角色偏移與類型偏移的分工

| 情況 | 主要報告 | 協調方式 |
|---|---|---|
| 角色不像自己 | 09_b | 09_f 可補充是否也造成類型偏移 |
| 角色仍像自己，但整場類型偏掉 | 09_f | 通知 09_b 無需重複命中 |
| 角色偏移造成作品偏移 | 09_b + 09_f | 兩份報告都命中，但理由分開 |

## 11. 與其他 QA 報告的協調

### 11.1 與 09_a

若 09_a 命中資訊說明腔，09_f 只判定它是否造成類型偏移。

例：

- 只是句子像工具提示：09_a 主報。
- 整場因此變成說明書式場景：09_f 也報。

### 11.2 與 09_b

若 09_b 命中角色變成系統、旁白或其他角色，09_f 需判定此問題是否同時改變作品類型氣質。

### 11.3 與 09_d

09_f 不判定「角色是否知道」，只判定「資訊揭露方式是否符合類型氣質」。

### 11.4 與 09_c

09_f 不做禁用詞機械比對。若某詞不在禁用詞表，但造成類型偏移，09_f 可命中；若只是不在詞表中允許，交給 09_c。

## 12. 修改建議格式

每個問題的修改建議必須具體到可執行層級。

| 問題等級 | 建議格式 |
|---|---|
| A 級 | 必修；指出要刪、壓縮、改成行動、改成沉默或改到別場 |
| B 級 | 建議修；指出替代方向 |
| C 級 | 邊緣案例；請人類確認是否保留 |

建議不得只寫「降低類型偏移」。

## 13. 檢查結果分級

| 等級 | 名稱 | 判定 |
|---|---|---|
| A | 高層偏移 | 整場方向偏離 00_b；通常 FAIL |
| B | 局部偏移 | 個別句子偏離；視密度 PASS / NEEDS_REVIEW / FAIL |
| C | 邊緣案例 | 可能是 intentional design；NEEDS_REVIEW |

### 13.1 偏移等級

| 偏移等級 | 定義 |
|---|---|
| 無 | 未發現偏移 |
| 局部 | 一到數句偏移，但不改變整場方向 |
| 高層 | 整場戲方向或收束方式偏移 |
| 嚴重 | 同時命中多個 00_b 核心基準，且會影響後續 QA 判定 |

## 14. Codex 使用規則

agent 執行本模板時：

1. 不得修改台詞檔內容，只能輸出 QA 報告。
2. 不得自行把 FAIL 改成 PASS。
3. 不得因句子寫得順就忽略 00_b 明文禁區。
4. 不得新增 00_b 沒有的類型偏移規則。
5. 不得把角色聲線問題與類型偏移問題混成同一條理由。
6. 不得產生 09_e。
7. 不得把 QA_PASSED 的台詞自動升 FINAL。
8. 若需要人類裁決，標 NEEDS_REVIEW 或 ARBITRATE_REQUIRED，不得替使用者拍板。

## 16. 最終原則

類型偏移檢查的核心不是判斷一句台詞好不好聽，而是判斷它是否把作品推向 `00_b` 明確排除或未授權的方向。

若台詞漂亮但方向錯，09_f 仍須標出。
