## 對抗式攻擊切角（先攻後判）

**Lens A（句型 / 抽象詞密度超標即 AI 味）攻擊：**
1. 攻：02_c §6 高風險句型掃描——「不是 A，而是 B」「我終於明白」「真正」「其實」「從今以後」是否出現？
   - 攻不破：諾拉「不是修辭，我計算過」表面含「不是」，但非「不是 A 而是 B」對稱主題宣言句式——是「否定修辭→改述為計算」的功能性切換，且立刻接條列數據，非工整對稱套語。全文無「我終於明白 / 真正重要的是 / 從今以後 / 其實我一直」。通過。
2. 攻：抽象詞密度——是否出現「守護 / 羈絆 / 命運 / 希望 / 重要 / 真正」等 02_c §3.1 抽象主題詞？
   - 攻不破：零命中。情感全部具象化為「加班費 / 連本帶利 / 三十秒 / 不可替換 / 處理效率 / 精神力」成本與效率語言。無抽象主題詞。
3. 攻：是否有角色替玩家朗讀整理劇情 / 設定說明（系統式旁白語氣）？
   - 攻不破：無。諾拉報數據是角色行為（戰術回報），非設定朗讀；無人解釋「我們的處境是…」式整理。
4. 攻：潛台詞是否被寫成明台詞（直接說在乎 / 害怕 / 謝謝）？
   - 攻不破：潛台詞全程遮掩——「謝謝你來」翻譯成「不可替換」、後怕翻譯成「差三十秒 / 加班費」、清道夫 OS「抖的是我」立刻刪掉。無明台詞化。
5. 攻：句子是否過度工整 / 禮貌 / 完整（AI 式漂亮句）？
   - 攻不破：句子帶髒話、破折號硬切、不完整（「下次再這樣，我就——（停）」）、口語碎句（「媽的，差三十秒」），反工整。

**Lens B（風格核心派 / 00_a §3.9.4）攻擊：**
1. 辯：諾拉的「系統回報語法」（防線剩三分四十 / 承載低於預估）會不會被機械 QA 誤判成「AI 助理 / 系統提示味」？

2. 辯：諾拉最長那段條列式（能…、能…、能…、而且…）是否 AI 式排比？
