# 1. 文件目的

本文件用於定義角色與角色之間的關係矩陣。

角色聲線不只由角色本人決定，也由他面對誰決定。  
同一個角色面對上位者、敵人、親密者、舊友、背叛者、陌生人時，語氣、稱呼、句長、攻防方式與沉默方式都可能不同。

本文件的目的，是讓 AI / Codex 在生成台詞時能判斷：

- 角色 A 面對角色 B 時是否會改變語氣？
- 兩人之間誰掌握權力？
- 兩人之間是否有情緒債、秘密、誤解或未說出口的事？
- 兩人如何稱呼彼此？
- 他們公開場合與私下場合說話有何不同？
- 哪些話他們絕不會對彼此說？
- 哪些稱呼或句子只在關係變化後才可出現？
- 關係是否已經進入新的階段？

本文件不是完整劇情大綱。  
詳細關係變化時間線應放入：

- `04B_關係變化時間線.md`

角色個人聲線應放入：

- `03B_主要角色聲線卡模板.md`
- `03_characters/main/角色名_聲線卡.md`

# 2. 關係管理原則

## 2.1 關係不是標籤

關係不應只寫成：

- 朋友
- 敵人
- 家人
- 戀人
- 師徒
- 搭檔
- 上司與下屬

這些只是表面標籤。  
有效的關係設計應包含：

- 表面關係
- 真實關係
- 權力差
- 情緒債
- 祕密
- 誤解
- 依賴
- 禁忌
- 稱呼變化
- 關係中的不能說出口
- 關係如何影響台詞

---

## 2.2 關係必須改變語言

若兩個角色的關係重要，它必須影響語言。

可能影響：

- 句長
- 稱呼
- 禮貌程度
- 攻擊方式
- 防禦方式
- 是否直視問題
- 是否使用真名
- 是否使用舊稱呼
- 是否使用黑話或暗號
- 是否能開玩笑
- 是否能道歉
- 是否能沉默
- 是否在公開場合偽裝

若關係不影響語言，該關係對台詞的作用可能不足。

---

## 2.3 關係應有不對稱性

好的關係通常不是雙方完全對等理解。

不對稱可以是：

- A 以為兩人是朋友，B 只把 A 當工具
- A 知道真相，B 不知道
- A 想保護 B，B 以為 A 看不起自己
- A 依賴 B，但不承認
- B 崇拜 A，但 A 對此感到厭惡
- A 已經原諒，B 還以為自己被恨
- A 對 B 有情緒債，但 B 不知道

關係不對稱會產生台詞縫隙，也是潛台詞的重要來源。

## 12.2 私密語言規則

私密語言不應一開始就解釋完。  
它應透過重複、變形、缺席與回收建立意義。

使用方式：

- 前期像普通話
- 中期玩家開始察覺特殊意義
- 後期在關鍵場景回收
- final 版不一定說破

# 16. Codex 使用規則

當 Codex 修改或使用本文件時，應遵守：

1. 不得只根據角色個人聲線生成關係場景，必須讀取關係矩陣。
2. 不得擅自修改 LOCKED 關係狀態。
3. 若台詞導致關係階段變化，需提醒更新 `04B_關係變化時間線.md`。
4. 若新增稱呼或暗號，需提醒更新 `02B_俗稱與黑話表.md`。
5. 若新增關係禁用詞，需提醒更新 `02C_禁用詞與慎用詞表.md`。
6. 若關係改變角色聲線，需提醒更新相關角色聲線卡。
7. 若關係變化影響主線，需提醒更新 `05C_角色弧線表.md` 或 `05D_資訊揭露表.md`。
8. 不得讓角色提前說破關係真相。
9. 修改後必須列出影響範圍。

- **基底類型（可組合）：** 合作（現場火力／威脅處理 × 資訊／自動化／能源技術）+ 上下級（功能性錯位：諾拉在資訊維「指揮」清道夫部署，清道夫在現場維對諾拉下達生存指令，無正式階級從屬）+ 救命（雙向：諾拉遠端哨戒砲／火控掩護救過清道夫，清道夫現場把被圍困的諾拉帶活）。非戀愛基底登記（成人線屬下游 §15 親密節奏，「樣本 → 指定偏好對象」基調見兩端聲線卡 §D，不在本關係類型層拍板）。<!-- INFERENCE: 親密＝研究式好奇轉指定偏好（清道夫 §D-2 第3 hook / 諾拉 §C-2 / §D-2），但本協議只建關係功能骨架，戀愛 / 親密階段不由本 skill 定；待人類確認是否在關係類型補登戀愛基底 -->
- **表面 vs 真實是否一致：** 不一致，且與前兩條關係的「演法」都不同——**雙方都不演（都不掩飾表面態度），但雙方都把真實動機翻譯成功能語言**。清道夫真的嫌她吵嫌她懶（不是裝的），但行動上完全依賴她的技術；諾拉真的把人換算成「處理單元 / 優先級」（不是裝冷），但她的價值排序已實質把清道夫列入高優先級。表面態度為真、底層在乎也為真，錯位在「兩人都缺乏（或拒絕）把在乎翻譯成情感語言的能力」。
- **哪一方在演：** 嚴格說兩人都不「演」（不偽裝表面關係），但兩人都**翻譯掉**真實動機——清道夫翻成成本／嫌麻煩，諾拉翻成優先級／效率。這是兩種「口是心非」語言系統的對撞，而非單方掩飾。

## 關係一句話

> 諾拉把「無名加工公司」當成一批通訊未完全上傳、被最低限度刪除／封存的殘片資料來解讀——她不憤怒、不控訴，只用系統回報式語言把這場區域蟲災逆向工程成「一條被打碎到沒人願意花錢拼回去的責任鏈」，於是這段關係的張力不在組織會不會反擊（它不會，它只是遺留物），而在諾拉以純功利／數據視角揭露真相時，**她自己拒絕承認的價值判斷**（這不是自然災害、有人該負責）正悄悄滲入她「萬物皆樣本」的冷處理。

- **對抗形式（角色對組織）：** 制度殘留 + 失控技術債 + 責任真空。ORG 是**事故源頭型對抗源**而非主動加害反派（ORG card §2）；它不回話，對抗體現在「殘留後果＋沒人負責」的被動結構。諾拉的「對抗」＝**解碼 / 拼接 / 判讀**，把被荒廢、蟲群、通訊不良與無人關心共同抹掉的資訊重新逆向出來（ORG card §4 資訊黑洞）。
- **ORG 端無回應提醒（下游）：** 本關係**永不**出現組織台詞 / 擬人化回話。所有「組織側」內容只能以**殘片文件語體**（成本報表官腔 / 黑市技術行銷話術 / 事故日誌 / 封存紀錄 / 解雇通知，ORG card §7 文件語體 hint）被諾拉**讀到 / 引用 / 反諷**，不得寫成 ORG 對諾拉說話。

## §2 權力差（角色對組織）

| 權力維度 | 誰握更多籌碼 | 說明 |

| 資訊（誰知道得更多） | **諾拉（解讀權）vs 組織（原始資料持有 → 已殘缺）** | 原始完整資料曾在組織手上，但經刪除／封存／斷線／蟲群佔據已成殘片（ORG card §4）；諾拉握的是**把殘片逆向拼接出真相的解碼能力**。即時權力在諾拉——她能讓沉默的殘片「開口」；但她受制於殘片本身的不完整（無法復原被徹底刪除的部分） |

| 社會地位 / 陣營位置 | 對等的「邊緣 / 無歸屬」 | 諾拉＝體制外流浪研究者；組織＝連名字都快被抹掉的小爛公司（ORG card §1）。兩者都在制度邊緣——一個被體制排擠，一個被體制放任爛尾 |
| 情緒主動權 | **諾拉（單向）** | 組織無情緒、不回應；情緒主動權全在諾拉一側。但諾拉的「主動權」是**冷處理式**的——她不對組織憤怒（憤怒不提高解析效率），只把它降格成「資料來源 / 失控案例樣本」 |

**權力翻轉情境（角色對組織）：**
1. 殘片不完整 / 被徹底刪除的部分，權力翻向「組織的遺留缺口」——諾拉再強的解碼也補不出不存在的資料，只能標「資料不足 / 因果鏈斷點」（呼應其聲線「路線資料不足。採用……的記憶」式承認缺口）。
2. 當殘片揭露的真相觸及「這不是自然災害、有人該負責」的價值判斷時，諾拉的純功利視角短暫被撬動——她仍會用數據語言包裝（「事故日誌顯示蟲群失控並非自然事件」），但揭露行為本身已隱含「該被揭露」的立場，是她難得不全然中立的時刻。
3. 真相若被解讀為「沒人願意花錢拼回去的責任鏈」（ORG card §2），權力回到「責任真空」——即使諾拉拼出全貌，也無對象可追責，組織以「已倒閉 / 跑路 / 無主」的方式讓追責落空。

## §6 不變的部分（關係恆定底線）

從本關係建立到主線結束，以下**絕對不變**（萃取自諾拉聲線卡不可偏移人格模板 + ORG card 不變量）：

1. 諾拉對組織遺留**永遠**用價值 / 優先級 / 效率 / 樣本邏輯解讀，**永不**情緒化控訴 / 義憤 / 抒情徒勞感（零抒情底線）。
2. 無名加工公司**永遠**是非人格遺留物——**永不**被寫成會回話 / 會反擊 / 會主動針對諾拉的反派 NPC；組織側只以殘片文件語體被引用（ORG card §7 / 00_l v0.3 §2.2）。
3. 對抗形式**永遠**是「制度殘留 / 責任真空 / 失控技術債」，**永不**降格成「諾拉 vs 一個壞老闆」的個人對決；事故源頭是責任鏈一環（最關鍵源頭環），不是孤立反派（ORG card §2）。
4. 蟲災真相**永遠**遵守資訊揭露順序——諾拉**永不**一加入就知全部真相，揭露隨殘片拼接進度推進（W-rules §22.1 / 諾拉聲線卡 §A）。
5. 諾拉的解碼**永遠**受殘片完整度限制——**永不**寫成「她能無中生有復原任何被刪除資料」的全知科技神（諾拉偏移基線 a.2）。
