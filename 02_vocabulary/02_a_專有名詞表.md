# 1. 文件目的

本文件是全作品專有名詞的正式索引。

本文件用於確保：

- 同一概念不會出現多個正式名稱
- 名詞不會在劇情尚未揭露前提前出現
- 角色不會使用自己不該知道的正式詞
- 台詞中不會過度堆砌世界觀名詞
- 官方名稱、俗稱、黑話、貶稱能夠分層管理
- Codex 與 AI 在生成台詞、任務包、劇情文件時能使用一致命名

本文件只管理「正式專有名詞」。

俗稱、黑話、貶稱、暗語、錯誤稱呼應放入：

- `02B_俗稱與黑話表.md`

禁用詞、慎用詞、解禁條件應放入：

- `02C_禁用詞與慎用詞表.md`

世界觀說明與背景設定應放入：

- `01A_世界觀總覽.md`

世界中人物如何說話，應放入：

- `01B_世界語言規格.md`
- `01C_陣營與階級語言.md`

# 2. 專有名詞管理原則

## 2.1 一個概念只能有一個正式名稱

每個核心概念應有唯一正式名。

可以有多個俗稱、黑話、貶稱、錯稱，但正式名稱只能有一個。

若出現多個候選正式名稱，應使用以下欄位決定：

| 候選名 | 優點 | 缺點 | 是否保留 | 備註 |
|---|---|---|---|---|
|  |  |  |  |  |
|  |  |  |  |  |

確定後，其餘名稱應移至 `02B_俗稱與黑話表.md` 或標記為 DEPRECATED。

---

## 2.2 正式名不等於角色一定會說

正式名稱是資料庫中的標準名稱，不代表角色在台詞中都會使用它。

角色是否使用正式名稱，取決於：

- 角色知識
- 階級
- 陣營
- 職業
- 教育程度
- 場景壓力
- 是否在公開場合
- 是否在偽裝
- 該名詞是否已對玩家揭露

例如：

| 名詞類型 | 正式文件中可用 | 角色台詞中可用 | 備註 |
|---|---|---|---|
| 官方組織名 | 是 | 依角色身份決定 | 平民可能只用俗稱 |
| 歷史事件真名 | 是 | 依揭露階段決定 | 前期可能禁止 |
| 超自然系統正式名 | 是 | 依教育與陣營決定 | 外行通常不會用完整名 |

---

## 2.3 名詞首次出現必須受控

每個重要名詞都應記錄：

- 首次在文件中出現的位置
- 首次在遊戲中對玩家出現的位置
- 首次可由角色說出口的位置
- 是否前期只能暗示
- 是否會構成劇透

若名詞本身暴露世界真相，必須與 `05D_資訊揭露表.md` 對齊。

# 17. Codex 使用規則

當 Codex 修改或建立本文件時，應遵守：

1. 不得擅自新增重大世界觀名詞，除非使用者明確提供或允許。
2. 若新增正式名詞，需同時填寫分類、簡短定義、首次出現與使用限制。
3. 若名詞有俗稱、貶稱、黑話，應提醒同步更新 `02B_俗稱與黑話表.md`。
4. 若名詞有禁用或解禁條件，應提醒同步更新 `02C_禁用詞與慎用詞表.md`。
5. 若名詞涉及世界本體設定，應提醒同步更新 `01A_世界觀總覽.md`。
6. 若名詞涉及台詞語感，應提醒同步更新 `01B_世界語言規格.md` 或 `01C_陣營與階級語言.md`。
7. 若名詞更名，應列出所有可能受影響文件。
8. 不得將 DEPRECATED 名詞重新用作正式名，除非使用者明確恢復。
9. 修改後必須列出修改檔案、改名項目與影響範圍。
