# 1. 文件目的

本文件用於建立次要角色與 NPC 的輕量化角色模板。

本文件的目標是平衡兩件事：

1. 不讓次要角色與 NPC 淪為中性說明機器。
2. 不為每個小角色建立過度沉重的主要角色聲線卡。

次要角色與 NPC 不一定需要完整心理弧線，但仍應具備：

- 明確場景功能
- 基本身份與陣營
- 語言特徵
- 資訊權限
- 可說與不可說內容
- 與玩家或主要角色的關係
- 場景壓力下的反應方式

本文件適用於：

- 單場出場角色
- 支線任務角色
- 商人、守衛、士兵、平民等功能型 NPC
- 章節限定角色
- 群體 NPC 類型
- 陣營代表小角色
- 可重複使用的對話模板角色
- 只承擔局部資訊或氣氛的角色

主要角色應使用：

- `03_characters/03_b_主要角色聲線卡模板.md`

# 2. 次要角色與 NPC 設計原則

## 2.1 NPC 也必須有立場

NPC 不應只是資訊容器。  
即使是功能型 NPC，也應受以下因素影響：

- 階級
- 地域
- 職業
- 陣營
- 恐懼
- 利益
- 偏見
- 是否信任玩家
- 是否被監視
- 是否有不能說出口的事

NPC 可以資訊少，但不應沒有立場。

---

## 2.2 NPC 不必完整，但要有角度

次要角色不一定需要完整人物傳記。  
但他至少應有一個說話角度。

可用角度包括：

- 怕惹麻煩
- 想賣東西
- 想討好上級
- 想保住家人
- 想隱瞞錯誤
- 看不起外地人
- 不相信官方說法
- 太相信官方說法
- 對玩家有舊恩怨
- 只相信錢
- 只相信規矩
- 想裝作什麼都不知道

沒有角度的 NPC，容易變成 AI 式說明文本。

---

## 2.3 功能型角色也需要語言差異

即使兩個 NPC 都是「守衛」，也應根據情境產生差異：

- 新兵守衛
- 老兵守衛
- 貪腐守衛
- 害怕的守衛
- 過度盡責的守衛
- 看不起玩家的守衛
- 暗中想幫玩家的守衛
- 被監視而不敢說真話的守衛

NPC 類型可以重複，但語氣應受場景與身份影響。

# 12. 可合併與降級規則

## 12.1 可合併條件

若兩個次要角色同時滿足以下條件，可考慮合併：

- 功能相同
- 所屬場景相近
- 沒有獨立弧線
- 沒有不同關係壓力
- 沒有明顯聲線差異
- 刪除其中一人不影響劇情

## 12.2 可降級條件

主要角色若不再承擔長期弧線，可降級為次要角色。  
次要角色若只提供一句資訊，可降級為：

- NPC 群體
- 環境文本
- 文件線索
- 旁人傳聞
- 道具描述
- 任務提示

# 13. Codex 使用規則

當 Codex 修改或建立次要角色 / NPC 文件時，應遵守：

1. 不得為每個小角色自動建立過重主要角色聲線卡。
2. 若角色承擔長期弧線，應提醒升級為主要角色聲線卡。
3. 若 NPC 提供世界觀資訊，需檢查其是否有資格知道。
4. 若 NPC 使用俗稱或黑話，需檢查 `02_vocabulary/02_b_俗稱與黑話表.md`。
5. 若 NPC 使用禁用詞，需檢查 `02_vocabulary/02_c_禁用詞與慎用詞表.md`。
6. 若 NPC 與主要角色有重要關係，需提醒更新 `04_relationships/04_a_角色關係矩陣.md`。
7. 若 NPC 可合併或刪除，應提出建議但不得擅自刪除。
8. 不得讓 NPC 替玩家整理劇情。
9. 不得讓 NPC 知道作者視角資訊。
10. 修改後必須列出影響範圍。

# 15. 最終原則

次要角色與 NPC 的目標，不是搶走主要角色的戲，而是讓世界有厚度、讓任務有角度、讓主要角色在互動中暴露自己。

好的 NPC 不需要完整人生，但需要：

- 一個立場
- 一個壓力
- 一種說話角度
- 一個不知道或不敢說的東西
- 一種受世界影響的語言

如果 NPC 只是把設定告訴玩家，他就不是角色，而是偽裝成角色的說明文字。
