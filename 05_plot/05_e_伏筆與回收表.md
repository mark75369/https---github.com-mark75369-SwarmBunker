# 1. 文件目的

本文件用於管理長篇遊戲劇本中的伏筆與回收。

伏筆不是單純提前放線索。  
有效伏筆應該在首次出現時自然成立，在後續劇情中逐步加深，並在回收時讓玩家重新理解前面的場景、物件、稱呼、台詞或行動。

本文件用於管理：

- 主線伏筆
- 角色伏筆
- 關係伏筆
- 世界觀伏筆
- 反派伏筆
- 結局伏筆
- 物件伏筆
- 稱呼伏筆
- 台詞回聲
- 場景細節回收
- 誤導與公平線索
- 分支伏筆與分支回收

本文件的目的是讓 AI / Codex 在生成大綱、章節、場景任務包與台詞時能判斷：

- 這個伏筆何時首次出現？
- 玩家當時應該如何理解？
- 玩家後來應該如何重新理解？
- 伏筆是否被中段加深？
- 回收時是否太直白？
- 回收是否由角色解釋過度？
- 伏筆是否與資訊揭露表一致？
- 伏筆是否被遺忘？
- 誤導是否公平？

# 3. 伏筆設計原則

## 3.1 伏筆首次出現時必須自然成立

伏筆不應像作者故意插入的提示。

首次出現時，它應該能以表面功能成立，例如：

- 一句日常台詞
- 一個角色習慣
- 一個地名
- 一個稱呼
- 一件小物品
- 一個任務限制
- 一個 NPC 誤稱
- 一個被角色避開的話題
- 一個看似普通的規矩

回收時，玩家才意識到它還有第二層意義。

---

## 3.2 回收不等於解釋

伏筆回收不應總是由角色說明：

- 原來當時那句話的意思是……
- 你還記得那個東西嗎？
- 其實那就是……
- 我早就留下了線索

更好的回收方式包括：

- 同一句話在新情境中重現
- 同一物件被不同人使用
- 舊稱呼突然變成真相
- 玩家看見物件位置改變
- 角色改口但不解釋
- 早期誤稱被證明接近真相
- 角色用沉默承認玩家已理解
- 任務結果讓早期細節重新變重

---

## 3.3 誤導必須公平

誤導不是欺騙玩家，而是讓玩家基於不完整資訊做出合理但錯誤的推論。

公平誤導應具備：

- 回頭看時線索存在
- 錯誤推論有角色或世界內理由
- 真相不是憑空出現
- 角色行為在真相揭露後仍然合理
- 沒有靠隱瞞鏡頭或作者作弊成立

不公平誤導包括：

- 真相揭露前完全沒有線索
- 所有人都用不自然方式避開關鍵資訊
- 角色為了騙玩家而違反自身動機
- 只靠結尾台詞補設定
- 早期內容與真相矛盾但未修正

# 4. 伏筆分類

## 4.1 伏筆類型表

| 類型 | 說明 | 常見載體 |
|---|---|---|
| 主線伏筆 | 指向核心真相、反派目的、世界謊言 | 事件、任務、敵人行動、官方說法 |
| 角色伏筆 | 指向角色過去、傷口、身份、選擇 | 反應、避談、物件、稱呼、習慣 |
| 關係伏筆 | 指向角色之間舊債、秘密、未說出口 | 稱呼、暗號、沉默、例外對待 |
| 世界觀伏筆 | 指向制度、歷史、地理、宗教真相 | 地名、禁忌、規矩、環境細節 |
| 反派伏筆 | 指向反派邏輯、計畫、弱點 | 行動模式、留下的後果、他人反應 |
| 結局伏筆 | 指向最終選擇與代價 | 道具、重複句、任務後果、角色價值觀 |
| 物件伏筆 | 某物件的用途或意義後期改變 | 道具、武器、信件、飾品、證件 |
| 稱呼伏筆 | 某稱呼後期揭露關係或身份 | 真名、舊稱、暱稱、頭銜、禁稱 |
| 台詞回聲 | 某句話後期以新意義重複 | 對白、口頭禪、誓言、反話 |
| 誤導伏筆 | 引導玩家形成錯誤但合理理解 | NPC 說法、文件、半真半假線索 |

## 7.2 首次出現注意事項

首次出現時應避免：

- 角色刻意提醒玩家記住
- 旁白或任務提示過度標記
- 畫面或台詞過度強調
- 使用太明顯的神秘語氣
- 讓角色說出「這以後會很重要」類似功能句

首次出現應像自然文本的一部分。

## 12.2 台詞回聲原則

台詞回聲不應只是重複名句。

有效回聲應做到：

- 相同句子在不同情境中意義改變
- 原本像玩笑，後來變成告別
- 原本像命令，後來變成信任
- 原本由 A 說，後來由 B 說，代表關係變化
- 原本是謊言，後來成為真心
- 原本是貶低，後來成為承認

應避免：

- 過度煽情重複
- 角色直接解釋這句話的意義
- 每個重要場景都回收同一句
- 回收時變成主題宣言

## 15.2 結局回收注意事項

結局回收應避免：

- 最後突然補設定
- 角色完整解釋前面所有伏筆
- 所有物件都強行回收
- 每條關係都有完整收尾
- 伏筆回收只服務驚訝，不服務情緒或選擇

結局回收應優先服務：

- 玩家選擇
- 角色代價
- 世界規則
- 情緒餘韻
- 前文再理解

## 16.2 分支補救原則

若玩家錯過伏筆，可使用：

- 不同 NPC 補充局部資訊
- 環境二次出現
- 任務後果重申
- 角色反應替代說明
- 可選文件或道具描述

但不應強行讓所有分支都得到完全相同資訊。  
分支差異應有代價。

# 17. 伏筆與台詞生成規則

## 17.1 生成台詞時的伏筆檢查

生成涉及伏筆的台詞前，應確認：

| 檢查項目 | 說明 |
|---|---|
| 伏筆狀態 | PLANNED / PLANTED / REINFORCED / PAID_OFF |
| 本場功能 | 首次放置 / 加深 / 誤導 / 回收 |
| 玩家應理解 | 本場玩家應理解到哪一層 |
| 角色是否知道 | 說話者是否知道真相 |
| 是否可說破 | 是否允許直接說出真相 |
| 是否有替代載體 | 能否用物件、沉默、稱呼或行動替代解釋 |

---

## 17.2 台詞中不應做的事

涉及伏筆時，台詞應避免：

- 直接解釋伏筆意義
- 提醒玩家「還記得嗎」
- 用過度神秘語氣標記線索
- 讓角色說出自己不該知道的回收
- 在首次放置時就暴露回收方向
- 回收時把所有前情整理成摘要

# 22. 最終原則

伏筆的核心不是讓玩家猜中，而是讓玩家在回收時感覺到：前面的世界一直是同一個世界，角色一直是同一個角色，真相一直埋在那裡。

好的伏筆應該：

- 首次出現時自然
- 中段加深時不急著說破
- 回收時改變玩家對前文的理解
- 服務角色、關係、世界或結局
- 不只服務驚訝
- 不靠角色朗讀答案

若伏筆只能靠角色在回收場景說明它是伏筆，代表伏筆本身還不夠成立。

伏筆不是裝飾。  
伏筆是長篇敘事讓玩家相信一切早有重量的方法。
