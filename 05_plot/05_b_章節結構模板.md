# 1. 文件目的

本文件用於管理長篇遊戲劇本的章節結構。

`05A_主線大綱模板.md` 定義整部作品的主線骨架；本文件則負責把主線拆成章節，明確定義每章：

- 為什麼存在
- 從哪裡開始
- 到哪裡結束
- 玩家在本章要做什麼
- 玩家在本章要知道什麼
- 玩家在本章仍不能知道什麼
- 哪些角色在本章改變
- 哪些關係在本章改變
- 哪些世界規則在本章被展示
- 哪些伏筆被放下或回收
- 哪些場景必須存在

本文件的目標，是避免章節變成單純任務清單或劇情流水帳。

章節不是一串事件。  
章節應該是一個可追蹤的變化單位。

# 3. 章節設計原則

## 3.1 每章必須有開始狀態與結束狀態

每章都應明確定義：

- 玩家進入本章時相信什麼
- 玩家離開本章時相信什麼
- 主角進入本章時想要什麼
- 主角離開本章時失去或得到什麼
- 主要關係在本章如何移動
- 世界壓力在本章如何升高或變形

若章節開始與結束沒有明顯差異，該章可能只是填充。

---

## 3.2 每章必須有不可替代功能

章節不可替代功能可以是：

- 引入主線核心問題
- 建立玩家初始誤解
- 第一次展示世界殘酷規則
- 迫使主角做出第一次不可逆選擇
- 讓主要角色關係從合作變成裂痕
- 揭露反派行動邏輯
- 讓玩家發現官方說法有漏洞
- 讓某個角色弧線進入下一階段
- 使某個稱呼、物件或承諾獲得新意義
- 推動結局條件

若一章刪除後，只少了一些事件，但主線、角色、資訊與關係都不受影響，該章功能不足。

---

## 3.3 每章應有主題壓力，但不應說成主題宣言

章節可以有主題壓力，例如：

- 信任的代價
- 秩序如何吞掉普通人
- 善意如何被制度扭曲
- 記憶與身份的衝突
- 生存與良知不可兩全

但章節台詞不應直接把這些主題說出來。  
主題應透過：

- 任務選擇
- 角色衝突
- 物件回收
- 關係變化
- 世界規則後果
- 玩家無法兩全的處境

呈現。

# 23. 最終原則

章節是長篇劇本的中層承重結構。

好的章節不是「發生了很多事」，而是讓玩家、角色、關係、世界與資訊狀態都從一個位置移動到另一個位置。

每一章都應能回答：

- 本章為什麼不可刪？
- 本章改變了誰？
- 本章揭露了什麼？
- 本章保留了什麼？
- 本章讓下一章為什麼必須發生？

若一章只提供任務量，卻沒有造成主線、角色、關係或資訊上的不可逆變化，該章尚未成立。
