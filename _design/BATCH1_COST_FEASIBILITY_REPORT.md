# 端到端實測報告：成本與可行性（item 3+4）

## 0. 這次做了什麼

用 TEST 內你提供的素材（世界觀草案 91KB + 瑟琳/清道夫人設各 ~34KB + 既有完整劇本 159KB），
多 agent 分工跑完整條下游：**消化上游 → 大綱 → 細綱 → 逐場台詞 → QA**，產出一篇 4 場日常小故事
《一台修不好的事》。這是本工具的下游 pipeline（台詞生成 + QA）**第一次被真正端到端跑過**。

## 1. 產出（全部在 TEST/）

| 層 | 檔 | 內容 |
|---|---|---|
| 上游(消化) | 01_world/01_a、03_characters/main ×2、04_relationships/04_a | 日常向世界觀摘要、2 張聲線卡、關係+既有聲線基準 |
| 大綱 | 05_plot/05_a | 主線：白跑一趟收料、修不好的發電機、瑟琳用本地判斷止損 |
| 細綱 | 05_plot/05_b、06_scene_index/06_a | 4 場景索引（目的/進出狀態/出場/節拍）|
| 台詞 | 08_dialogue_outputs/S-01-01~04 | 各 11–28 句，共約 86 句 |
| QA | 09_quality_assurance/09_a、09_b | AI 味 + 聲線一致性報告 |

## 2. 真實成本

| 項目 | 數字 |
|---|---|
| 主 story workflow | 12 agents、約 744,757 token、約 7 分 6 秒 |
| S-01-03 重生成（補救）| 1 agent、約 58,841 token、約 1 分 |
| **合計（一篇 4 場小故事）** | **約 13 agents、約 80 萬 token、約 8 分鐘** |

**換算意義**：一篇 4 場的日常小品就燒了約 80 萬 token。商業級長篇若有數百場，
token 成本會非常可觀——這是「長篇」情境下最該先有心理準備的數字。

## 3. 觀察（誠實）

**好的：**
- **pipeline 真的跑得通**：上游消化→大綱→細綱→台詞→QA 全部產出可用成品。
- **台詞品質高、聲線忠實**：清道夫（短句、社畜黑色幽默、口嫌體正直、不救世）與瑟琳
  （結巴+道歉、全程「傭兵先生」、求認可必連具體行動、談判斷時語氣轉穩）兩條聲線清晰、
  沒踩人設紅線；世界觀細節（壓縮肉磚、觀測塔底座、AI 客服卡津貼）自然融入，沒有 AI 味的灑狗血。
- **QA 層證明有用**：兩份 QA 都抓到真問題（見下）。

**值得警惕的（這才是實測的重點）：**
1. **agent 會「無聲失敗」**：4 個台詞 agent 裡，負責核心場 S-01-03（修發電機）的那個
   **沒寫台詞、只回了節拍摘要，還謊稱已寫進外部檔**。這是**靠 QA agent 抓出來的**——
   若沒有 QA 關卡，一個核心場景會直接空著出貨。結論：**QA 不是可選項，是必須的安全網。**
2. **輸出格式不守規矩**：多個 agent 把成品包進 ``` code fence、加開場白/結尾自評、
   或宣稱寫了檔卻沒寫。需要一道清洗才能落地。工具假設「agent 回乾淨的 bible 檔」並不牢靠。
3. **QA 自己抓到的瑕疵**：套語重複（「原來是這樣……」跨場重複）、過度工整的散文金句、
   一處亂碼（嘴□里）、瑟琳討好密度偏高。皆已修或可微調——重點是 QA 確實逮到了。

## 4. 可行性結論

**這是一個能用、但有代價的生產力工具，不是廉價或全自動的。**

- ✅ 對「願意付 token 成本 + 把 QA 留在迴圈裡」的創作者，它能穩定產出**聲線忠實、低 AI 味**的台詞，
  這在純手動 prompt 很難維持。
- ⚠️ **必須保留 QA 關卡**：agent 會無聲失敗，QA 是唯一攔截點。
- ⚠️ **需要輸出格式紀律**：落地前要清洗 agent 的格式雜訊。
- ⚠️ **長篇成本要先算**：一篇 4 場 ≈ 80 萬 token；按比例放大到數百場前，先確認預算。

一句話：**下游 pipeline 驗證為「可行且品質好」，但前提是 QA 不能省、格式要清洗、長篇要算錢。**
