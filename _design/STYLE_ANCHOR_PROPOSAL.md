# STYLE_ANCHOR_PROPOSAL — 台詞生產前置文風錨定機制設計提案

# 0. 文件用途

本提案處理一個 user 在 10th master 第十輪 pre-Milestone-4 階段提出的下游台詞生產問題：

> **「生成台詞時要先經過一個規則限制，避免 agent 產出台詞會優先使用該模型的預設寫法，而讓角色台詞與前文的文筆不同。」**

關鍵在 **pre-generation guard**（前置守門），不是 post-hoc QA。要在 agent 寫台詞之前就鎖死規則，避免它套用模型預設文風、讓新台詞跟前文文筆斷裂。

本提案範圍：

- 問題陳述與 gap 分析
- 從既有腳本（`台詞/Script_Level_07/08/09.tsv`）量化抽取的文風指紋
- 三層敘事規範（角色對話 / 主角第一人稱旁白 / 環境描述）
- 4 角色（清道夫 / 瑟琳 / 莉娜 / 諾拉）聲線樣本師設計
- 檔案 schema（`00_b §1.x` 擴充 + `01_world/01_d` 新建）
- Pipeline 整合點（透過 `07_a` 任務包 §18 風格要求）
- 給主開發線（10th master CODEX batch 流程）的 acceptance criteria

本提案 **不直接動 `00_b`、`01_world/`、`07_scene_tasks/`、`.claude/skills/`**；只產設計 RFC。實作由後續 batch 落地。

# 1. 問題陳述

## 1.1 user 觀察到的具體現象

user 已寫過 3 個 Level 的腳本（`台詞/Script_Level_07/08/09_adjusted.tsv`，共 171 行），這些是該作品的「前文文筆」基準。

當 agent 用 `/dialogue-write` 生成新台詞時，user 擔心：

1. agent 會優先套用 LLM 預設文風（過度禮貌、過度完整、抽象詞堆疊、主題宣言）
2. 新台詞的句長、標點、口頭禪、旁白風格會與前文斷裂
3. 三層敘事（角色對話 / 主角內心 OS / 第三方描述）會混同或漂移

## 1.2 為什麼這不只是「AI 味問題」

`00_protocol/00_b_反ai味檢查表.md` §8-§24（通用 QA 層）已覆蓋大量 AI 味反模式（抽象詞、宣言句、心理直說、潛台詞被說破）。但 user 的需求**超出 AI 味檢查範圍**：

| 需求 | AI 味檢查可解決？ | 落差 |
|---|---|---|
| 避免抽象詞 / 主題宣言 / 心理直說 | ✅ 09_a / 00_b §10-§13 已覆蓋 | 無 |
| 角色去名後可辨識 | ✅ 09_b / 00_b §17.1 已覆蓋 | 無 |
| 句長分佈匹配前文 | ❌ 無量化基線 | **缺** |
| 標點密度（省略號、軟硬停）匹配前文 | ❌ 無量化基線 | **缺** |
| 旁白用第一人稱「我們」視角 | ❌ 無敘事人稱規範 | **缺** |
| 主角內心 OS 寫在旁白層而非對話層 | ❌ 無三層結構規範 | **缺** |
| 各角色的特徵詞 / 句型指紋 | ⚠ 03_b 聲線卡定性描述，無量化指紋 | 部分缺 |

**結論：** AI 味檢查是「通用反模式」，本提案是「作品特定指紋錨定」。兩者互補，不可替代。

## 1.3 為什麼後置 QA 不夠

POST-hoc QA（`/qa` 8 份報告，含 09_b 角色聲線一致性）能抓出問題，但：

1. agent 已經寫了一版「LLM 預設文風」的台詞 — 修稿成本高
2. QA 只能標問題，不能教 agent 「該怎麼寫」
3. 多次來回 trial / converge / QA 之間累積偏移，難以收斂

**Pre-generation guard 的價值：** 把指紋直接餵給 agent 作為硬約束，讓 trial 版本一開始就在正確軌道上。

## 4.6 旁白模式

來自 4 角色合計 43 行旁白樣本：

- **視角**：第一人稱複數「我們」（"留下我們面面相覷" / "讓我們都不禁倒吸一口氣"）
- **內容**：物理動作 / 環境變化 / 設備運作 / 移動 — **絕無心理標籤**（"她沉默地凝視" "他心中湧起酸楚" 等通用文藝旁白）
- **動作描述邊界**：允許「皺眉看完」（外顯動作 + 結果），不允許「她憂愁地皺著眉，想起了從前」（心理直說 + 回憶插入）
- **句長**：mean=27.3（顯著長於對話），上限 43 字
- **諾拉旁白特例**：極短功能性（「打哈欠」3 字）— 諾拉旁白只描述她的物理動作，不負責環境

# 5. 三層聲線規範草案

## 5.1 層 A：角色對話「」

### 通用規則

1. 用「」（直角引號）標示，不用 “”（彎引號）
2. 句子內標點用全形（，。！？…）
3. 不用破折號（——）
4. 對話本體不含括號旁白；如需動作補充，另起一行（）

### Per 角色硬約束（量化）

| 角色 | 平均句長 | 中位數 | 標點特徵 | 字典 |
|---|---|---|---|---|
| 清道夫 | 11-14 字 | ~11 | 軟硬停均衡；省略號偶爾用 | 吐槽詞 + 命令短句 + 同伴名 |
| 瑟琳 | 6-11 字 | ~6 | **省略號高密度（1.5-2 / 行）**；硬停 > 軟停 | 「傭兵先生」固定；驚訝詞「那個…」「她…她」 |
| 莉娜 | 11-14 字 | ~13 | 硬停為主；少省略號 | 罵字（哼、嘖、混帳）+ 攻擊性稱呼 + 技術簡稱 |
| 諾拉 | 10-15 字（含 36+ 例外）| ~10 | 軟硬停皆密；省略號偶爾用 | 17 個技術詞 + 數據口吻 + 邏輯連接詞 |

## 5.2 層 B：主角第一人稱旁白（）

**僅限男主角清道夫使用**（其他角色旁白比例 < 5%，不適用此層規範）。

### 規則

1. 視角：第一人稱複數「我們」（首選），第一人稱單數「我」（次選），絕無第三人稱
2. 句長：mean 25-30 字，上限 45 字
3. 內容類型：
   - 移動 / 場景描述
   - 設備啟動 / 環境變化
   - 通訊 / 物件交接
   - 群體反應（"我們面面相覷"）
4. 禁用：
   - 心理標籤（「他憂鬱地」「她悲傷地」）
   - 回憶插入（「想起多年前」）
   - 主題暗示（「彷彿命運的安排」）
   - 全知敘事（「殊不知這只是開始」）
5. 允許邊界案例：
   - 物理動作 + 結果（「皺眉看完」「踹開最後一隻蟲子」）
   - 客觀觀察（「動作快得驚人」）— 從觀察者角度，不從被觀察者內心

### 內心 OS 處理

主角的內心話（OS）**寫在旁白層**，不寫在對話層。但 OS 不寫成「他心想」，而是寫成第一人稱觀察 / 評價：

✅ 正例：`（諾拉這女人絕對是我見過最莫名其妙的傢伙。）`
❌ 反例：`「諾拉這女人絕對是我見過最莫名其妙的傢伙。」`（OS 不應寫成口說對話）
❌ 反例：`（他心想：諾拉這女人絕對莫名其妙。）`（不寫「他心想」）

## 5.3 層 C：環境 / 第三方描述（）

**被 5.2 涵蓋** — 主角旁白即此層。本作目前無獨立全知敘述者，不需要再分一層。

如未來加入序章 / 終章 / 章節間 cut-scene 需全知視角，再以本提案 v0.x 修訂版另立規範。

## 5.4 跨層硬約束

1. **三層比例維持**：對話 60-70% / 旁白 20-25% / 其他 10-15%
2. **角色去名測試**：拿掉 `MainCharacter`/`MainGirlA/B/C` label 後，仍能從句長 + 標點密度 + 特徵詞分辨四角色
3. **旁白歸屬**：旁白優先歸主角；其他角色旁白比例 < 5%（諾拉短行為例外）
4. **零破折號** —— 永久禁用

# 6. 4 角色聲線樣本師設計

每位角色在 `01_world/01_d_文風樣本與指紋.md` 含：
