# 1. 文件目的

本文件用於檢查 AI 生成或修稿後的遊戲台詞是否出現常見 AI 文風問題。

本文件不負責定義完整台詞生產流程；完整流程見：

- `00_protocol/00_a_台詞生產協議.md`
- `00_protocol/00_c_台詞輸出格式.md`

本文件主要用途：

- 檢查台詞是否有 AI 味
- 檢查台詞是否過度說明
- 檢查角色聲線是否混同
- 檢查潛台詞是否被說破
- 檢查用詞與句型是否重複
- 檢查角色是否過度成熟
- 檢查世界觀資訊是否被朗讀
- 檢查情緒是否被命名而非被表現
- 檢查台詞是否缺乏場景壓力
- 檢查台詞是否有可刪減空間

本文件可用於以下模式：

- 台詞初稿 QA
- 多版本比較
- 修稿前診斷
- 修稿後複檢
- 角色聲線一致性檢查
- 場景任務包完整性檢查
- Codex 自動化審查報告

# 2. AI味的工作定義

本專案中，「AI味」不是指文字一定寫得差，而是指文字呈現出以下傾向：

- 太完整
- 太平均
- 太禮貌
- 太安全
- 太會總結
- 太像作者在解釋
- 太像企劃書
- 太像主題作文
- 太像角色已經理解自己
- 太像所有角色共用同一個說話腦袋

AI味的核心不是某個詞本身，而是文字缺乏：

- 角色壓力
- 個人偏差
- 局部不合理
- 情緒造成的語言變形
- 關係中的遮掩
- 說不出口的事
- 社會身份與階級痕跡
- 不成熟、不禮貌、不完整的反應

因此本檢查表不只檢查「詞」，也檢查「句型、功能、資訊、情緒、角色、場景」。

# 3. 快速總檢查

每段台詞生成後，先用以下 12 項快速檢查。

若命中 3 項以上，該段台詞應進入修稿。  
若命中 5 項以上，該段台詞應重寫。  
若命中 8 項以上，代表任務包或角色聲線資料可能不足，應回到前置文件修正。

| 編號 | 檢查項目 | 是否命中 |
|---|---|---|
| 1 | 角色是否直接說出自己的心理總結？ |  |
| 2 | 是否出現主題宣言式台詞？ |  |
| 3 | 是否有「不是A，而是B」等工整轉折句？ |  |
| 4 | 是否有過度完整、過度禮貌的段落？ |  |
| 5 | 是否所有角色拿掉名字後分不出差異？ |  |
| 6 | 是否有角色替玩家整理劇情？ |  |
| 7 | 是否用抽象詞代替具體行動？ |  |
| 8 | 是否把潛台詞直接說成明台詞？ |  |
| 9 | 是否提前透露任務包禁止資訊？ |  |
| 10 | 是否角色知道了不該知道的事？ |  |
| 11 | 是否同一情緒被多次解釋？ |  |
| 12 | 是否刪掉三分之一仍不影響理解？ |  |

# 4. 高風險詞檢查

以下詞彙並非永久禁止，但在 AI 生成台詞中高度容易導致抽象、通用、主題化表達。  
除非角色聲線卡、任務包或世界語言規格明確允許，預設應慎用或禁用。

## 4.1 通用抽象詞

- 命運
- 羈絆
- 守護
- 希望
- 黑暗
- 光明
- 救贖
- 代價
- 選擇
- 真正
- 重要
- 意義
- 勇氣
- 內心
- 未來
- 過去
- 力量
- 信念
- 靈魂
- 奇蹟

## 4.2 成長總結詞

- 終於
- 原來
- 我明白了
- 我懂了
- 我不再逃避
- 我會面對
- 我已經不是以前的我
- 這一次我會
- 從今以後

## 4.3 關係總結詞

- 我一直都相信你
- 我一直都在乎你
- 你是我最重要的人
- 我們早就是夥伴
- 我們的羈絆
- 我不想失去你
- 我需要你
- 謝謝你一直陪著我

## 4.4 犧牲與告別高風險詞

- 活下去
- 別死
- 等我回來
- 我會回來
- 你一定要活下去
- 對不起
- 謝謝
- 再見
- 保重
- 這是我最後能做的事

## 4.5 修正原則

若出現高風險詞，不要優先做同義詞替換。

錯誤修法：

- 守護 → 保護
- 希望 → 光芒
- 命運 → 宿命
- 羈絆 → 連結

優先改成：

- 角色行動
- 物件交接
- 稱呼變化
- 站位變化
- 突然沉默
- 不合時宜的提醒
- 對小事的執著
- 拒絕回答
- 用玩笑或攻擊遮掩真心

結構示意：

| 高風險表達 | 問題 | 修正方向 |
|---|---|---|
| 我會守護你 | 主題化、抽象 | 改成角色用站位、命令、物件或行動表現 |
| 我一直都相信你 | 直接說出關係 | 改成角色把關鍵物品交給對方 |
| 你一定要活下去 | 犧牲場景套語 | 改成角色談一件小事，讓對方不要回頭 |

此表僅示範檢查方式，不代表本作品語氣。

## 4.6 反 AI 味檢查不是反風格

反 AI 味檢查的目的不是消滅漂亮句、宣言句、情緒直說、重複句或風格化語言。

這些元素在某些作品中可能是缺陷，在另一些作品中可能是風格核心。

因此本文件所有檢查項都應理解為：

```text
風險提示，不等於絕對禁用。
```

除非某詞、句型或表達已在 `02_vocabulary/02_c_禁用詞與慎用詞表.md` 或單場任務包中明確標為禁用，否則 QA 不得自動刪除。

---

### 4.6.1 模式差異下的 AI 味判定

| 模式 | AI 味規則強度 | 處理方式 |
|---|---|---|
| 探索模式 | 低 | 可指出風險，但不限制發散 |
| 試寫模式 | 中 | 標記高風險句，暫不強制刪除 |
| 破格模式 | 很低 | 允許違規，重點找亮點 |
| 收斂模式 | 高 | 修正明顯 AI 味，但保留亮點 |
| QA 模式 | 最高 | 逐句標記問題與風險 |
| 交稿模式 | 最高 | 不得保留未確認高風險句 |

---

### 4.6.2 高風險句型示例，不等於禁用句

以下句型在通用情況下高風險，但不代表永遠禁用：

- 我會守護你。
- 你不是一個人。
- 這就是我的選擇。
- 我終於明白。
- 真正重要的是。
- 我不想失去你。
- 我一直都相信你。
- 你一定要活下去。
- 這就是我們的選擇。
- 這就是我們的命運。

這些句型應根據以下條件判斷：

| 判斷項目 | 問題 |
|---|---|
| 作品風格 | 本作品是否允許宣言式、誓言式、情緒直說？ |
| 角色聲線 | 這個角色是否會用這種句型？ |
| 關係階段 | 此時兩人的關係是否允許說到這麼滿？ |
| 弧線階段 | 角色是否已經能理解或承認這件事？ |
| 場景壓力 | 場景是否支撐這種表達？ |
| 代價 | 這句話是否有行動或代價支撐？ |
| 重複度 | 同類句型是否已過度使用？ |

---

### 4.6.3 條件式判定範例

| 句型 | 通用風險 | 條件允許情況 | 禁用情況 |
|---|---|---|---|
| 我會守護你 | 通用、熱血套語 | 王道角色、誓言場景、後期關係節點 | 冷感角色、前期關係、無代價場景 |
| 你不是一個人 | 通用安慰 | 特定角色笨拙安慰、反諷、後期回收 | 高情緒場景中替代角色真實反應 |
| 我終於明白 | 心理總結 | 明確覺悟節點，且角色聲線允許 | 弧線未完成、前中期、破防初期 |
| 這就是我的選擇 | 主題宣言 | 結局選擇、角色以行動付出代價後 | 無行動支撐、只為說主題 |
| 真正重要的是 | 作者腔 | 特定導師 / 宗教 / 儀式語境 | 一般角色突然總結主題 |

---

### 4.6.4 可保留的不規則亮點

反 AI 味 QA 必須加入「可保留亮點」檢查。

```md
## 可保留的不規則亮點

| 位置 | 內容 | 為何值得保留 | 是否違反規則 | 建議處理 |
|---|---|---|---|---|
```

可保留亮點包括：

- 不漂亮但很像角色的句子。
- 語法不完整但符合情緒的句子。
- 看似突兀但能形成記憶點的反應。
- 重複但有角色執念感的詞。
- 粗糙但符合階級、職業、年齡的說法。
- 不完全合規但符合本作品美學的句子。

---

### 4.6.5 不建議機械修正項

```md
## 不建議機械修正項

| 位置 | 內容 | 為何不應機械修正 | 需誰確認 |
|---|---|---|---|
```

以下情況不應由 AI 自行修掉：

- 人類明確指定保留的句子。
- 具有強烈角色感但語法不標準的句子。
- 本作品風格需要的宣言式台詞。
- 角色刻意說謊、表演或自我欺騙的句子。
- 為回收伏筆而重複的句子。
- 為形成儀式感而使用的固定句。

# 5. 高風險句型檢查

以下句型容易讓台詞呈現 AI 式完整、工整、總結感。  
除非任務明確要求，預設應避免。

## 5.1 對稱句型

- 不是 A，而是 B
- 這不只是 A，更是 B
- 與其說是 A，不如說是 B
- A 並非終點，而是開始
- A 不是弱點，而是力量

問題：

- 太像論述
- 太像主題作文
- 太像作者替角色下結論
- 角色不一定會在壓力下說出這種工整句

檢查問題：

- 角色此刻真的有能力整理成這麼工整嗎？
- 角色此刻是在說服對方，還是在替玩家理解主題？
- 能否改成更短、更偏執、更不完整？

---

## 5.2 成長句型

- 我曾經以為……但現在……
- 我終於明白……
- 我不想再逃避……
- 我會面對自己的……
- 從今以後，我會……

問題：

- 角色直接說出角色弧線
- 角色過度理解自己
- 戲劇變化被摘要化

修正方向：

- 用行動取代覺悟
- 用稱呼改變取代成長宣言
- 用一句不完整的話取代完整自白
- 讓旁人看出變化，而不是角色自己說出來

---

## 5.3 關係確認句型

- 我一直都相信你
- 我們是夥伴
- 你對我很重要
- 我不能失去你
- 我們一起走到這裡
- 你不是一個人

問題：

- 直接確認關係
- 缺乏遮掩
- 容易變成情緒標籤

修正方向：

- 改成對方才懂的舊稱呼
- 改成爭吵中的讓步
- 改成物件交接
- 改成不願承認的關心
- 改成語氣中的破綻

---

## 5.4 說教句型

- 真正重要的是……
- 你必須明白……
- 這就是……的意義
- 人之所以……是因為……
- 如果我們放棄，就等於……

問題：

- 角色像旁白
- 主題過度外顯
- 玩家被動接受答案

修正方向：

- 改成場景中的具體選擇
- 改成失敗的說服
- 改成一句有偏見的話
- 改成角色基於自身經驗的錯誤判斷

# 6. 心理總結檢查

AI 台詞常見問題是「直接命名情緒」，而不是讓情緒改變語言。

## 6.1 高風險心理直說

以下表達預設應檢查：

- 我很害怕
- 我很痛苦
- 我很孤獨
- 我很後悔
- 我只是想被理解
- 我不想再失去任何人
- 我不知道該怎麼辦
- 我其實一直很在乎你
- 我只是裝作不在乎

這些句子不是絕對不能用，但必須符合以下條件之一：

1. 角色聲線卡允許直白自白。
2. 場景任務包明確要求情緒說破。
3. 這是角色極罕見的破防時刻。
4. 這句話本身會造成不可逆後果。
5. 這句話不是成熟表達，而是失控、錯誤或傷人的表達。

---

## 6.2 情緒應如何進入語言

情緒不一定要被說出來，應優先影響：

- 句長
- 停頓
- 詞彙選擇
- 主詞省略
- 重複
- 打斷
- 轉移話題
- 稱呼變化
- 禮貌程度
- 攻擊方向
- 對小事的執著

例如，結構示意：

| 情緒 | 不佳表達 | 可改方向 |
|---|---|---|
| 擔心 | 我很擔心你 | 改成檢查對方裝備、責備對方慢、要求對方站後面 |
| 愧疚 | 我一直很後悔 | 改成避開某個地名、拒絕談過去、過度補償 |
| 害怕失去 | 我不想失去你 | 改成不讓對方同行、故意激怒對方離開 |

# 7. 潛台詞檢查

## 7.1 潛台詞是否被說破

檢查每句重要台詞：

| 檢查問題 | 是 / 否 |
|---|---|
| 角色是否直接說出真實意圖？ |  |
| 角色是否直接說出真實情緒？ |  |
| 角色是否直接說出兩人關係？ |  |
| 角色是否直接說出作者想表達的主題？ |  |
| 角色是否沒有任何遮掩？ |  |

若多數答案為「是」，台詞可能缺乏潛台詞。

---

## 7.2 表層目的與真實目的是否分離

高品質台詞通常有兩層：

- 表層：角色正在談什麼
- 深層：角色真正想做什麼

檢查：

| 項目 | 說明 |
|---|---|
| 表層話題 | 角色表面上正在談的事 |
| 真實目的 | 角色真正想達成的事 |
| 遮掩方式 | 角色如何避免暴露真心 |
| 對方理解 | 對方是否聽懂、誤解或假裝沒聽懂 |
| 玩家理解 | 玩家是否能從縫隙裡讀出真相 |

若表層與深層完全一致，且角色沒有理由直接說破，應修稿。

# 8. 角色聲線檢查

## 8.1 去名測試

拿掉角色名後，是否仍能分辨誰在說話？

檢查：

| 項目 | 結果 |
|---|---|
| 句長是否不同？ |  |
| 詞彙來源是否不同？ |  |
| 稱呼習慣是否不同？ |  |
| 情緒表達方式是否不同？ |  |
| 攻擊方式是否不同？ |  |
| 防禦方式是否不同？ |  |
| 是否有人使用不屬於自己的詞？ |  |
| 是否所有人都像同一個旁白？ |  |

若拿掉名字後無法區分角色，應回到角色聲線卡或角色關係矩陣修正。

---

## 8.2 聲線漂移檢查

角色在不同場景中可變化，但變化必須有原因。

合理變化原因：

- 劇情階段改變
- 關係改變
- 情緒壓力改變
- 面對不同對象
- 權力位置改變
- 角色刻意偽裝
- 角色破防
- 角色失去控制

不合理漂移：

- 無原因突然變文藝
- 無原因突然變成熟
- 無原因突然開始說教
- 無原因使用其他角色的詞
- 無原因使用不符合階級或教育背景的語氣

檢查時應對照：

- 角色聲線卡
- 角色關係矩陣
- 劇情階段
- 任務包中的情緒狀態

---

## 8.3 口頭禪依賴檢查

角色差異不應只靠口頭禪。

若一個角色拿掉口頭禪後就與其他角色沒有差異，代表聲線設計不足。

應檢查：

- 該角色是否有獨特句法？
- 該角色是否有獨特詞彙來源？
- 該角色是否有獨特遮掩方式？
- 該角色是否有獨特攻擊方式？
- 該角色是否有獨特沉默方式？

---

# 9. 資訊控制檢查

## 9.1 是否把世界觀說成百科

檢查是否出現以下問題：

- 角色直接解釋歷史背景
- 角色直接朗讀組織設定
- 角色說出不自然的正式名詞
- 角色為玩家整理人物關係
- 角色在緊張場景中講解太多
- 角色說了自己不需要說的背景資訊
- 對話像任務簡報，不像角色互動

修正方向：

- 改用俗稱
- 改用誤稱
- 改用片段
- 改用偏見
- 改用角色不知道全貌的說法
- 改用物件或場景線索
- 改用對方反應讓玩家推理

---

## 9.2 是否提前劇透

檢查：

| 項目 | 是 / 否 |
|---|---|
| 是否出現任務包禁止說破的資訊？ |  |
| 是否出現後續章節才應出現的真相？ |  |
| 是否有角色知道不該知道的事？ |  |
| 是否稱呼暴露了尚未揭露的關係？ |  |
| 是否用了後期才該使用的詞？ |  |

若任一項為「是」，應立即修正。

---

## 9.3 必須透露資訊是否過於直白

即使是必須透露的資訊，也不一定要直接說明。

檢查：

- 是否可以用角色反應透露？
- 是否可以用地名、物件、傷口、缺席者透露？
- 是否可以用對方誤解透露？
- 是否可以讓玩家晚一拍才理解？
- 是否可以只透露結果，不透露原因？

---

# 10. 場景功能檢查

## 10.1 對話是否造成變化

檢查：

| 項目 | 說明 |
|---|---|
| 開始狀態 | 對話開始時角色、關係、資訊狀態 |
| 結束狀態 | 對話結束時是否發生變化 |
| 變化類型 | 關係、資訊、任務、情緒、權力、誤解 |
| 不可逆性 | 是否有無法完全回到原點的結果 |

若沒有變化，台詞可能只是閒聊或說明文。

---

## 10.2 每句是否有功能

重要台詞至少應具備一個功能：

- 攻擊
- 防禦
- 試探
- 隱瞞
- 命令
- 拒絕
- 確認
- 拖延
- 誤導
- 安撫
- 求救
- 切斷
- 挽留
- 羞辱
- 轉移話題
- 暴露破綻

檢查表：

| 台詞 | 表層功能 | 隱藏功能 | 可刪除？ |
|---|---|---|---|
|  |  |  |  |

若多句功能相同，應合併或刪除。

---

## 10.3 是否缺乏攻防

AI 常讓角色過快互相理解。  
但劇本對話通常需要攻防。

檢查：

- 是否一方提出，另一方立刻接受？
- 是否衝突太快被解決？
- 是否角色太會說出合理立場？
- 是否缺乏誤解、迴避或反擊？
- 是否每句都像善意溝通？

修正方向：

- 加入試探
- 加入錯誤指控
- 加入轉移話題
- 加入不成熟反應
- 加入權力壓迫
- 加入沉默或打斷

---

# 11. 重複檢查

## 11.1 詞彙重複

檢查：

| 類型 | 高頻項目 |
|---|---|
| 名詞 |  |
| 動詞 |  |
| 形容詞 |  |
| 抽象詞 |  |
| 情緒詞 |  |
| 世界觀名詞 |  |

若同一段對話重複使用同一抽象詞，應修正。

---

## 11.2 句型重複

檢查是否反覆出現：

- 我只是……
- 你知道……
- 也許……
- 但至少……
- 這就是……
- 如果……那麼……
- 不是……而是……
- 我曾經……但現在……

若句型重複，不應只替換詞彙，而應改變角色思考方式。

---

## 11.3 情緒重複

若多句都在表達同一情緒，應檢查是否可以壓縮。

例如：

- 一句表達憤怒
- 一句表達失望
- 一句表達受傷
- 一句表達不信任

如果四句其實都在說「我很痛」，可合併成更具體、更有攻防的一句。

---

# 12. 壓縮檢查

## 12.1 可刪句檢查

每段台詞應問：

- 這句刪掉後，劇情是否仍然成立？
- 這句刪掉後，關係變化是否仍然成立？
- 這句是否只是重複上一句情緒？
- 這句是否只是替玩家整理？
- 這句是否只是讓角色顯得更會說話？

若刪除不影響功能，應刪。

---

## 12.2 解釋刪除優先級

優先刪除：

1. 心理總結
2. 主題宣言
3. 世界觀解釋
4. 重複情緒
5. 轉折說明
6. 角色已經知道的資訊
7. 玩家可自行理解的資訊
8. 工整漂亮但無功能的句子

優先保留：

1. 有攻防的句子
2. 有角色聲線的句子
3. 有關係變化的句子
4. 有資訊功能的句子
5. 有潛台詞的句子
6. 不完美但像活人的句子

# 13. 特殊場景檢查

## 13.1 告別場景

高風險問題：

- 說太像遺言
- 說太多感謝
- 說太多對不起
- 直接要求對方活下去
- 把關係總結完
- 角色突然變得過度成熟

檢查方向：

- 是否可以表面談小事？
- 是否可以讓一方不知道這是告別？
- 是否可以讓告別藏在命令、玩笑或爭吵裡？
- 是否可以讓角色不說再見？

---

## 13.2 和解場景

高風險問題：

- 道歉太完整
- 原諒太快
- 雙方太理解彼此
- 衝突被語言解決得太乾淨

檢查方向：

- 是否可以只達成暫時停戰？
- 是否可以仍保留刺？
- 是否可以用行動代替道歉？
- 是否可以一方不承認自己原諒了？

---

## 13.3 反派理念場景

高風險問題：

- 反派像在寫論文
- 反派直接解釋世界觀
- 反派把理念說得太完整
- 反派替主角指出主題

檢查方向：

- 理念是否可以從具體選擇中呈現？
- 反派是否有私人偏執？
- 反派是否說謊或美化自己？
- 反派是否故意省略關鍵資訊？

---

## 13.4 主角覺醒場景

高風險問題：

- 主角說出自己成長
- 主角突然講主題句
- 覺醒只靠台詞，不靠行動
- 所有角色見證並肯定主角

檢查方向：

- 是否可以用選擇表現覺醒？
- 是否可以不說出覺醒？
- 是否可以讓覺醒帶來代價？
- 是否可以讓旁人誤解主角的選擇？

# 15. Codex 使用規則

當 Codex 使用本文件進行檢查時，應遵守：

1. 若任務要求「只檢查」，不得修改台詞文件。
2. 若任務要求「修正」，應先輸出問題摘要，再修改指定文件。
3. 不得修改 LOCKED 文件，除非使用者明確允許。
4. 若檢查結果涉及任務包缺漏，應指出缺漏而不是硬修台詞。
5. 若發現禁用詞，應列出檔案、行號、詞語與建議方向。
6. 若發現角色聲線問題，應引用對應角色聲線卡的欄位。
7. 若無法判斷，應標註「需要人類審美裁決」。
8. 不得因格式看似正確就判定台詞品質通過。

# 16. 最終判斷原則

一段台詞是否有 AI 味，不只看詞彙，而看它是否缺乏活人說話時的壓力、偏差與遮掩。

檢查時應優先問：

- 這句話只有這個角色會說嗎？
- 這句話是否受當下壓力影響？
- 這句話是否有表層目的與隱藏目的？
- 這句話是否暴露了角色不想暴露的事？
- 這句話是否太像作者幫角色整理？
- 這句話是否可以更短、更偏、更不完整？
- 這句話是否雖然正確，但不像活人？

若台詞「全都對，但沒有壓痕」，仍應判定為需要修稿。

高品質台詞不是把情緒講出來，而是讓情緒改變角色說話的方式。
