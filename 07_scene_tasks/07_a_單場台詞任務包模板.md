# 1. 文件目的

本文件是正式生成單場台詞前的最重要輸入格式。

單場台詞任務包的目的，是把一場戲所需的全部上下文壓縮成 AI / Codex 可執行的任務指令，避免生成台詞時出現以下問題：

- 角色聲線漂移
- 台詞像作者旁白
- 台詞提前劇透
- 關係進度錯誤
- 角色突然成熟或和解
- 場景功能不明
- 台詞只是在交換資訊
- 世界觀名詞使用錯誤
- NPC 知道不該知道的事
- 對話沒有攻防與潛台詞
- 高情緒場景變成 AI 式總結
- 生成結果格式不一致

本文件回答：

- 這場戲是什麼？
- 這場戲為什麼存在？
- 這場戲開始時各角色狀態是什麼？
- 這場戲結束時必須改變什麼？
- 哪些資訊必須說、只能暗示、禁止說破？
- 角色此時的聲線、關係、弧線與稱呼狀態是什麼？
- 這場戲要產出哪種台詞版本？
- 生成後要用哪些 QA 規則檢查？

任務包不是完整世界觀資料，也不是角色卡全文。  
它是單場生成所需的最小高密度上下文。

# 3. 任務包使用原則

## 3.1 任務包只放本場必要資訊

任務包不應複製整份世界觀、整份角色卡或整份大綱。

應只放：

- 本場需要知道的設定
- 本場出場角色的必要聲線摘要
- 本場相關關係狀態
- 本場可用與不可用資訊
- 本場台詞限制
- 本場輸出要求

不應放：

- 與本場無關的大量歷史
- 後期才會揭露的真相
- 未出場角色的完整設定
- 不會影響台詞的背景百科
- 作者自己知道但角色不知道的答案

---

## 3.2 任務包必須明確限制 AI

任務包不是只告訴 AI 要寫什麼，也要告訴 AI 不能寫什麼。

必須明確定義：

- 禁止透露資訊
- 禁止使用詞彙
- 禁止出現的稱呼
- 禁止讓角色完成的成長
- 禁止關係過快變化
- 禁止變成的文風
- 禁止加入的劇情
- 禁止自動補完的內容

限制越清楚，台詞越不容易變成通用 AI 劇本。

---

## 3.3 任務包應先定義場景功能，再要求台詞

台詞是完成場景功能的手段，不是場景的目的。

每個任務包都應先回答：

- 這場戲必須完成什麼？
- 角色在這場戲中彼此要爭什麼？
- 玩家看完這場後應多知道什麼或誤解什麼？
- 關係是否改變？
- 伏筆是否放下或回收？

若這些答案不清楚，不應直接生成正式台詞。

# 4. 任務包文件頭格式

每份單場任務包應使用以下文件頭：

```md
# CHxx_Sxx_台詞任務包

狀態：DRAFT / REVIEW / LOCKED / DEPRECATED
版本：v0.1
最後更新：YYYY-MM-DD
適用範圍：CHxx_Sxx
優先級：高

若本文件與舊聊天紀錄衝突，以本文件為準。
```

## 4.1 狀態定義

| 狀態 | 意義 | AI / Codex 可做事項 |
|---|---|---|
| DRAFT | 草稿任務包 | 可指出缺漏、建議補充、產出試寫 |
| REVIEW | 已可試寫 | 可生成多版本，但需保留 QA |
| LOCKED | 已鎖定任務包 | 不得擅自更改條件，只能按任務生成 |
| DEPRECATED | 已棄用 | 不得作為正式台詞依據 |

# 22. Codex 使用規則

當 Codex 修改或使用單場任務包時，應遵守：

1. 不得擅自修改 LOCKED 任務包核心條件。
2. 不得在任務包缺漏時直接生成 final 台詞。
3. 若任務包與上游文件衝突，必須回報衝突。
4. 若生成台詞需要新增設定，必須先標註為 DRAFT，不得寫入正式設定。
5. 若生成台詞導致關係變化，需提醒更新 `04_relationships/04_b_關係變化時間線.md`。
6. 若生成台詞導致資訊揭露變化，需提醒更新 `05_plot/05_d_資訊揭露表.md`。
7. 若生成台詞新增伏筆或回收，需提醒更新 `05_plot/05_e_伏筆與回收表.md`。
8. 若生成台詞新增稱呼、黑話或禁用詞，需提醒更新 `02_vocabulary/02_b_俗稱與黑話表.md` 或 `02_vocabulary/02_c_禁用詞與慎用詞表.md`。
9. 若台詞已可保存，需使用 `08_dialogue_outputs/08_b_生成台詞檔案模板.md` 格式。
10. 修改後必須列出影響範圍。

# 25. 最終原則

單場台詞任務包的目標，是讓 AI 在寫台詞前沒有藉口亂猜。

好的任務包應該讓生成者清楚知道：

- 這場戲要完成什麼
- 角色此時是誰
- 角色此時知道什麼
- 角色此時不能知道什麼
- 角色此時想遮掩什麼
- 角色此時如何稱呼對方
- 角色此時不能說哪些漂亮但錯誤的話
- 台詞生成後要如何判斷成敗

若任務包足夠清楚，AI 生成台詞時就不需要靠通用文風補洞。  
而高品質台詞的核心，正是不要讓 AI 用通用文風補洞。
