# 2. 場景一句話

> 本場表面上是諾拉搬進基地、接管自動化與能源系統，實際上要讓「諾拉 ↔ 清道夫」從實驗室初遇的臨時休戰，轉成「功利期的入隊」——諾拉用數據語言把整支隊伍評估成有「維護價值」的資產、把基地當成「有電有牆能睡的地方」，清道夫嘴上嫌她把基地當公園佔用、手上卻已經默許她接管系統；並避免本場變成兩人互相理解的暖心入隊大會，或諾拉提前解碼蟲災真相。

# 3. 場景功能

## 3.1 本場必須完成

1. 把諾拉從「實驗室的外人」實質接進灰鴉基地，落地她的功能定位＝資訊／自動化／能源大腦（CH-06「四人成形」的技術拼圖補齊；05_b §24.1 CH-06）。
2. 用諾拉的「維護價值／優先級」功利語言，把入隊動機定調成「這裡是目前最有探索價值、最好睡、外部威脅處理效率最高的據點」，而非情感歸屬（諾拉功利期；05_b §24.3 矩陣 C-諾拉 CH-06）。
3. 埋下伏筆：基地自動化＝後段聯合防衛戰的基礎（05_b §24.4 CH-06「埋：基地自動化＝後段防衛戰基礎，回收 CH-09」；05_e §0）。

## 3.2 本場不能變成

1. 不能變成兩人互相理解、暖心接納的入隊大會（諾拉零抒情、清道夫去英雄化雙底線）。
2. 不能變成諾拉的技術／世界觀說明會（她只回報判斷有價值的部分，不主動解說設定；諾拉聲線污染檢查 §7）。
3. 不能讓諾拉提前透露蟲災真相、組織責任或蟲后存在（CH-08 才解禁；05_d §0）。
4. 不能變成清道夫直說「歡迎加入／有妳真好」式的接納（去英雄化偏移基線）。
5. 不能把兩人的功能語言寫成真冷漠（互相需要藏在成本／優先級術語下；04_a §R-清道夫-諾拉 關係一句話）。
