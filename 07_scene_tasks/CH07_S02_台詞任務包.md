# 7. 場景一句話

> 本場表面上是清道夫衝進蟲群包圍區把實驗失誤陷險的諾拉拖出來的一次救援，實際上要讓兩人從「互為功能性高價值資產」往「不可替換的人」推進一格——諾拉用數據式「你目前不可替換」這句包裝過的話遮掩剛剛差點沒命的後怕與在乎，清道夫用成本 / 加班費記帳遮掩「我其實在乎妳會不會出事」，並避免任何一方把這份在乎翻譯成情感語言（避免本場最大風險：救援煽情化 / 提前說破關係 / 提前洩漏蟲災真相）。

# 8. 場景功能

## 8.1 本場必須完成

1. 演出救援動作：諾拉因實驗失誤 / 殘渣設備故障被蟲群圍困，清道夫冒險衝入把她帶出（建立「武力 / 現實接口維清道夫高」與「諾拉生活自保趨近零、需被救」的權力差落地，04_a §2）。
2. 落地諾拉弧線「不可替換期」：諾拉以數據 / 優先級語言（「你目前不可替換」「重新找一個同等耐用、高處理效率的人很麻煩」變體）說出對清道夫的依賴，全程零抒情、口是心非（05_c 諾拉 CH-07；聲線卡 §B-9 後期語氣）。
3. 推進 R-清道夫-諾拉 情緒債「被否認 + 加深中」：清道夫在諾拉真有生命危險後漏半句在乎、立刻用成本語言撤回（04_a §5 第 2 條破戒情境）。

## 8.2 本場不能變成

1. 不能變成救援煽情大會（清道夫不可說「我不能失去妳 / 我會保護妳」；違去英雄化偏移基線）。
2. 不能變成諾拉被嚇哭 / 求救 / 情緒外露的戲（違諾拉偏移基線 a.4：初登場核心＝被圍困卻想睡，壓力下更冷更數據化、不崩潰）。
3. 不能變成兩人互相理解大會 / 把關係說滿（「不可替換」是諾拉的包裝詞，不可被任一方翻譯成「我們相愛 / 你對我很重要」）。
4. 不能變成世界觀說明會 / 提前洩漏蟲災真相（蟲災=人為養殖技術外洩屬 CH-08 才解禁，本場禁止由任一角色台詞觸及，05_d §0）。
5. 不能讓諾拉變成有問必答的服務型 AI / 劣質語言模型（聲線污染底線 §7）。
6. 不能讓清道夫變成救世主 / 聖父；照顧只能藏在命令 / 嫌棄 / 記帳。
