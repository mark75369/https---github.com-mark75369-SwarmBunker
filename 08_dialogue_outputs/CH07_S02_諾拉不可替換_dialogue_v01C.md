## 本場功能摘要

本場一句話：情緒推到最滿、但兩人都死守不直說的版本——清道夫的後怕快要漏出來，每次都被他用更兇的成本語言壓回去；諾拉察覺到他的後怕，於是把「不可替換」說得比平常更具體、更長，那是她能給的最接近「謝謝你來」的話。

本場必須完成：
1. 演出救援動作（落地權力差，04_a §2）。
2. 落地諾拉「不可替換期」數據式半承認依賴（05_c CH-07）。
3. 推進情緒債「被否認 + 加深中」（04_a §5 第 2 條）。

本場不能變成：
1. 救援煽情大會 / 清道夫直說保護。
2. 諾拉抒情 / 求救 / 情緒崩潰。
3. 關係說滿。
4. 提前洩漏蟲災真相（05_d §0）。

> 本版方向（v01C）：情緒更重，但避免直說。清道夫的後怕用「更兇的罵 + 反覆檢查她有沒有受傷的動作」漏出再壓回；諾拉句子拉得比平常長、更具體（系統回報語法的「過度精確」即是她的情緒），但全程零抒情。情緒在動作與潛台詞層，不在台面字句。

## 台詞正文

（蟲群密度異常升高。殘渣設備連環過載、火光竄起，自布防禦失效。諾拉被崩落的支架壓住一條退路，蟲群正從另一端湧入。）

諾拉：（被困，聲音平穩到不像在險境）防線剩三分四十。已扣除我這側的兩具殘骸阻擋係數。蟲群會在崩潰後轉向最近熱源。

（她頓了極短一拍。）

諾拉：……熱源是我。這項我無法自行修正。

（清道夫的腳步聲幾乎是撞進來的，連開三槍，把逼近的蟲打退。）

清道夫：（喘得很重，吼）妳給我說一次！一次就好！「我碰了不該碰的東西」——很難嗎！

諾拉：我碰了不該碰的東西。

清道夫：（卡了半秒，撥開殘骸）……現在才聽話。手。

（他抓住她，另一手反覆按過她的肩、手臂，像在確認。）

清道夫：哪裡傷了。

諾拉：無外傷。你不必檢查第三次。

清道夫：（沒停手，低吼）閉嘴。跟緊，貼著我，別亂動。

（蟲群壓上。清道夫把她整個人擋在身後退出，火力全開。一段最緊的撤離。塌方封住通道，蟲群被隔開。）

清道夫：（單膝撐地，喘）……出來了。

（他沒馬上起來。背對著她，肩膀還在起伏。）

諾拉：（看著他的背，停頓）你的呼吸頻率異常。

清道夫：（猛地站起，背過身換彈，聲音壓得很硬）跑那麼快當然喘。少廢話。這筆加班費，這次我一毛都不會少算。

諾拉：……

清道夫：（頓）下次再這樣，我就——（停）……別碰沒檢查的東西。聽到沒有。

（長停頓。諾拉看著他僵硬的背影。）

諾拉：（很慢，很具體）你目前不可替換。不是修辭。我計算過。能在三分四十內到達、能單人壓制這個密度的蟲群、能把我從崩塌側帶出、而且願意回來的單元——只有一個。

清道夫：（沒回頭，握槍的手緊了一下）……

諾拉：所以你不能死。重新找一個同等耐用、高處理效率的人，很麻煩。

清道夫：（很久，才開口，聲音放回嫌棄）……「很麻煩」。妳可真會說話。

諾拉：先去睡了。剛剛做了一堆預期外的事，浪費我不少精神力。細節等我睡醒再談。

（她搖搖晃晃往內走。清道夫看著她，直到她消失在轉角。）

清道夫：（OS）她說她計算過。

清道夫：（OS）我也計算過。如果晚三十秒——（停）……算了。

清道夫：（OS）加班費照算。後面那句，刪掉。

## 設計註解

### 聲線註解

- 清道夫：後怕用「更兇的罵 + 反覆檢查傷勢的動作」漏出（「妳給我說一次！」「哪裡傷了」「你不必檢查第三次」），每次漏出立刻用成本 / 嫌棄壓回（「跑那麼快當然喘」「加班費一毛不少」），破折號處「下次再這樣，我就——」硬生生切斷改口（§16.2 回合 3 漏半句立刻撤回），OS「後面那句，刪掉」自我封口。守去英雄化：全程不說保護 / 在乎 / 不能失去。對齊 §B BeforeBattle_014/021/024 + 偏移基線。
- 諾拉：情緒以「系統回報語法的過度精確」表現——句子比平常長、更具體（「能在三分四十內到達…願意回來的單元，只有一個」），這份反常的詳盡本身即是她的後怕與在乎，但字面零抒情、零告白。收尾要睡迴避。對齊 §B 第 3/9/11 列 + 聲線一句話「情緒越強越用更冷靜的數據語言遮掩」。

### 關係註解

- 情緒債「被否認 + 加深中」最濃版本：雙方都碰到真心邊緣（清道夫的喘 / 諾拉的過度精確）、各自撤退（清道夫改口、諾拉要睡）。對齊 §16.2 回合 4。
- 關係推進一格但未命名、未說破；諾拉式半破戒「說結果不說因」極大化（把「謝謝你來」翻譯成最長的效率計算）。

### 資訊控制註解

- 諾拉只報防線 / 阻擋係數 / 威脅戰術數據，不碰蟲災成因（§13.3 / a.2）。
- 「不可替換」說得最具體但仍是效率語言，未翻譯成情感（§13.4 / §16.3）；守住 02_c §3.3 紅線（無「我需要你 / 你很重要」）。
- 救援險境不寫成瀕死遺言場（§18.3）：諾拉不訣別、不求救，清道夫不說告別套語。

### 伏筆 / 回收註解

- 「能源 / 設備連環過載＝前置檢查隨便」回收（§15.1）。
- 「不可替換」以一次具體救援事件 + 過度精確的效率計算驗證，不用宣言（§15.1）。
