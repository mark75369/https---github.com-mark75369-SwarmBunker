## 本場功能摘要

本場一句話：清道夫衝進蟲群包圍區把實驗失誤陷險的諾拉拖出來，諾拉用一段過度精確的效率計算把「謝謝你來」翻譯成「你目前不可替換」，清道夫用加班費與一句被切斷的關心撤回後怕——「不可替換」被一次具體救援事件而非任何宣言驗證。

本場必須完成：
1. 演出救援動作（落地武力 / 現場接口維清道夫高、諾拉自保趨近零的權力差，04_a §2）。
2. 落地諾拉「不可替換期」：以數據 / 優先級語言半承認依賴，全程零抒情、口是心非（05_c CH-07）。
3. 推進 R-清道夫-諾拉 情緒債「被否認 + 加深中」（04_a §5 第 2 條）。

本場不能變成：
1. 救援煽情大會（清道夫不可直說保護 / 不能失去）。
2. 諾拉被嚇哭 / 求救 / 情緒外露（違偏移基線 a.4）。
3. 兩人互相理解大會 / 把關係說滿。
4. 世界觀說明會 / 提前洩漏蟲災真相（05_d §0，本場禁觸）。

## 台詞正文

（蟲群密度異常升高。殘渣設備連環過載、火光竄起，自布防禦失效。諾拉被崩落的支架壓住一側退路，蟲群正從另一端湧入。）

諾拉：（被困，聲音平穩到不像在險境）防線剩三分四十。蟲群會在崩潰後轉向最近熱源。

（她頓了極短一拍，順手用斷管撥開撲上的幼蟲，動作不慌。）

諾拉：……熱源是我。這項我無法自行修正。

（清道夫的腳步幾乎是撞進來的，連開三槍把逼近的蟲打退。）

清道夫：（喘得很重，吼）又是妳！說過多少次，別碰沒檢查過的東西！這次又拆了什麼？

諾拉：能源耦合模組。我試圖把輸出提高。承載低於預估。

清道夫：（撥開殘骸）承載低於預估——所以妳把整條防線玩到三分鐘崩，自己當蟲子的點心？

諾拉：因果關係上來說，是的。

清道夫：（一把抓住她）狗娘養的，妳就不能怕一次嗎？

諾拉：恐慌不會提高脫困效率。

清道夫：（拽著她，另一手反覆按過她的肩、手臂）……哪裡傷了。

諾拉：無外傷。你不必檢查第三次。

清道夫：（沒停手，低吼）閉嘴。跟緊，貼著我，別亂動。

（蟲群壓上。清道夫把她整個人擋在身後退出，火力全開。一段最緊的撤離。塌方封住通道，蟲群被隔開。）

清道夫：（單膝撐地，喘）……出來了。媽的，差三十秒。

（他沒馬上起來。背對著她，肩膀還在起伏。）

諾拉：（站定，撣灰）三十秒在可接受範圍。

清道夫：（猛地站起，背過身換彈，聲音壓得很硬）可接受？妳的「可接受」每次都要老子加班把妳從蟲嘴裡撈回來。這筆加班費，連本帶利。

諾拉：記在帳上。

清道夫：（頓）下次再這樣，我就——（停）……別碰沒檢查的東西。聽到沒有。

（長停頓。諾拉看著他僵硬的背影。）

諾拉：（很慢，很具體）你目前不可替換。不是修辭，我計算過。能在三分四十內到達、能單人壓制這個密度、能把我從崩塌側帶出、而且願意回來的單元——只有一個。

清道夫：（沒回頭，握槍的手緊了一下）……

諾拉：所以你不能死。重新找一個同等耐用、高處理效率的人，很麻煩。

清道夫：（很久，才開口，把聲音放回嫌棄）……「很麻煩」。妳這張嘴，倒是會挑最不像人話的方式說。

諾拉：最精確的方式。先去睡了——剛剛做了一堆預期外的事，浪費我不少精神力。細節等我睡醒再談。

（她搖搖晃晃往內走。清道夫看著她，直到她消失在轉角。）

清道夫：（OS）她說她計算過。三十秒，她連抖都沒抖。

清道夫：（OS）抖的是我。

清道夫：（OS）……加班費照算。後面那句，刪掉。

## 設計註解

### 聲線註解

- 清道夫：危機切短命令（「閉嘴。跟緊，貼著我，別亂動」），罵綁具體標的、髒話有對象有功能（「狗娘養的，妳就不能怕一次嗎」「媽的，差三十秒」），後怕用「反覆檢查傷勢的動作 + 更兇的罵」漏出，每次立刻用成本 / 嫌棄壓回；破折號「下次再這樣，我就——」硬切改口（§16.2 回合 3 漏半句立刻撤回）；OS「抖的是我」「後面那句，刪掉」自我封口。全程守去英雄化（無保護 / 在乎 / 不能失去直說）。對齊 §B BeforeBattle_003/014/021/024 + AfterBattle_005 + 髒話來源欄。
- 諾拉：系統回報語法（「防線剩三分四十」），認錯不道歉冷靜歸因（「因果關係上來說，是的」「恐慌不會提高脫困效率」），情緒以「系統回報語法的過度精確」表現（最長那段「不可替換」計算），用最冷的效率帳把「謝謝你來」翻譯掉，收尾要睡迴避。零粗口、不慌、不求救。對齊 §B 第 3/9/10/11 列 + 聲線一句話「情緒越強越用更冷靜的數據語言遮掩」。

### 關係註解

- 攻防四回合對齊 §16.2：① 短命令救人 + 嫌不管前置 / 冷靜報數據；② 救出記帳「連本帶利」/ 接受功利框架「記在帳上」；③ 嘴軟漏在乎（破折號改口 + 反覆檢查傷勢）立刻撤回 / 「不可替換」過度精確包裝；④ 雙方各自撤退（清道夫放回嫌棄 + OS 封口 / 諾拉要睡）。
- 「錯位的冷處理對撞」：清道夫攻擊全被諾拉冷處理吸收，諾拉以過度精確 +「你不能死」反將。R-清道夫-諾拉 從「高價值人力支援」往「值得維護的對象」推進一格，不命名、未說破（04_a §R-清道夫-諾拉）。

### 資訊控制註解

- 諾拉只報能源 / 防線 / 威脅戰術數據，不碰蟲災成因（§13.3 / 知情矩陣 a.2）。
- 清道夫罵停在「爛攤子 / 加班費」現況層，不碰組織知情真相（§13.3）。
- 「不可替換 / 別死 / 很麻煩」皆功利語言，未翻譯成情感直說；守住 02_c §3.3 紅線（無「我需要你 / 你很重要 / 我不能失去妳」）（§14.3）。
- 救援險境不寫成瀕死遺言場：諾拉不訣別、不求救，清道夫不說告別套語（§18.3 / 02_c §3.4）。

### 伏筆 / 回收註解

- 「能源輸出過載＝諾拉前置檢查隨便」缺陷後果回收（§15.1）。
- 「彼此不可替換」主題以一次具體救援事件 + 過度精確的效率計算驗證，不用宣言說破（§15.1 / §15.2）。

### 不可被 QA 磨平的亮點（§19.2.4，收斂保留）

- 諾拉用最功利的話（過度精確的「不可替換」計算 + 「你不能死」）做最在乎的事——口是心非反差萌。
- 清道夫漏半句在乎立刻用成本語言撤回的「嘴推手做」節奏（破折號改口 + OS「後面那句，刪掉」）。
- 兩種口是心非語言系統（成本 vs 優先級）的對撞。

### 待人類確認（09_e final-gating）

- 中高風險救援 + 不可替換主題 + 真相禁區邊界需人類確認例外與定稿（§19.2.5）。
- 「連本帶利」黑色幽默濃度、清道夫破折號改口節奏、諾拉最長那段計算的精確度，是否照保留至 FINAL。
