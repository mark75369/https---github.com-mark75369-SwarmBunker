## 本場功能摘要

本場一句話：救援險境下兩人話更密、攻防更尖——清道夫一邊救一邊狠罵她不管安全前置，諾拉幾乎不接招、用數據歸因與功利換算把每一記攻擊吸收掉，最後用「不可替換」反將一軍，把在乎包裝成最冷的效率帳。

本場必須完成：
1. 演出救援動作（落地權力差，04_a §2）。
2. 落地諾拉「不可替換期」數據式半承認依賴（05_c CH-07）。
3. 推進情緒債「被否認 + 加深中」（04_a §5 第 2 條）。

本場不能變成：
1. 救援煽情大會。
2. 諾拉情緒外露 / 求救。
3. 關係說滿 / 互相理解大會。
4. 提前洩漏蟲災真相（05_d §0）。

> 本版方向（v01B）：攻防更強、衝突更尖。清道夫罵得更密更狠（仍綁具體標的、不惡毒），諾拉以無視 / 數據歸因 / 重複前句正面接住每一拳；句長中等，攻防回合清楚（§16.2 四回合）。

## 台詞正文

（蟲群密度異常升高。殘渣設備過載炸出火光，自布防禦失效。諾拉被塌落的支架與湧上的蟲群夾在中間。）

諾拉：（被困，平板）防線剩三分五十。蟲群轉向最近熱源。

（清道夫破門衝進來，連開兩槍把撲近的蟲轟散。）

清道夫：（吼）又是妳！說過多少次，別碰沒檢查過的東西！這次又拆了什麼？

諾拉：能源耦合模組。我試圖把輸出提高。承載低於預估。

清道夫：（撥開殘骸）承載低於預估——所以妳把整條防線玩到三分鐘崩，自己當蟲子的點心？

諾拉：因果關係上來說，是的。

清道夫：（一把抓住她）狗娘養的，妳就不能怕一次嗎？

諾拉：恐慌不會提高脫困效率。

清道夫：（拽著她貼牆退）……閉嘴。跟緊，別讓我回頭找妳。

（蟲群壓上，清道夫單手壓火力、單手把她往後拖。一段密集的撤退動作，槍聲與崩塌聲交織。）

清道夫：（喘）出來了。媽的，差三十秒。

諾拉：（站定，撣灰）三十秒在可接受範圍。

清道夫：（轉身指她鼻子）可接受？妳的「可接受」每次都要老子加班把妳從蟲嘴裡撈回來！這筆加班費，連本帶利！

諾拉：記在帳上。利息部分，請附計算式。

清道夫：（噎住）……妳。

諾拉：你目前不可替換。

清道夫：（瞪她）妳又來這套。

諾拉：陳述事實。重新找一個同等耐用、能在三分鐘內撈我出來、又罵不死的人，成本過高、效率過低。所以——別死。

清道夫：（一愣，隨即冷笑，重新裝彈）……「別死」。妳這張嘴，倒是會挑最不像人話的方式說。

諾拉：最精確的方式。

清道夫：（背過身）行。那妳也省著點，別再給我把防線玩崩。

諾拉：先去睡了。利息細節等我睡醒再談。

（她飄回內側。清道夫盯著她的背影，半晌。）

清道夫：（OS）三十秒。她連抖都沒抖。

清道夫：（OS）抖的是我。……這筆，不記帳上。

## 設計註解

### 聲線註解

- 清道夫：攻擊更密更狠但綁具體標的（「別碰沒檢查過的東西」「把整條防線玩到三分鐘崩」），髒話頻率上探有對象有功能（「狗娘養的」「媽的」），危機切短命令（「閉嘴。跟緊」），關心藏進連本帶利的加班費，OS 漏一句「抖的是我」立刻自我撤回（不記帳上）。對齊 §B BeforeBattle_003/014/024 + 髒話來源欄。
- 諾拉：正面接住每一拳但全用數據歸因（「因果關係上來說，是的」「恐慌不會提高脫困效率」），重複前句式吸收（「記在帳上。利息部分，請附計算式」），用最冷的效率帳包裝「別死」。零粗口、零慌張。對齊 §B 第 3/7/9/10 列 + 偏移基線 a.4。

### 關係註解

- 攻防四回合對齊 §16.2：① 短命令救人 + 嫌不管前置 / 冷靜報數據；② 救出記帳 / 接受功利框架；③ 嘴軟漏在乎立刻撤回 / 「不可替換」包裝；④ 被功利話戳到岔開 / 收尾要睡。
- 「錯位的冷處理對撞」最尖版本：清道夫攻擊力道全被諾拉冷處理吸收，反被「別死」將一軍。不命名、未說破（04_a §R-清道夫-諾拉）。

### 資訊控制註解

- 諾拉只報能源 / 防線 / 威脅戰術數據，不碰蟲災成因（§13.3 / 知情矩陣 a.2）。
- 清道夫罵停在「爛攤子 / 加班費」現況層，不碰組織知情（§13.3）。
- 「別死」「不可替換」皆功利語言，未越界成情感直說（§14.3 紅線守住）。

### 伏筆 / 回收註解

- 「能源輸出過載＝諾拉前置檢查隨便」回收（§15.1）。
- 「不可替換」以反將一軍的效率帳驗證，不用宣言（§15.1 / §15.2）。

### 待人類確認

- 「連本帶利 / 利息」黑色幽默濃度是否合 user 喜好（§19.2.3 允許冒險範圍內）。
